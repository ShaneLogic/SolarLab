What lives in the project tree?

This decides the data model: single-device vs multi-device, whether runs are persistent objects, whether sweeps are first-class.

A

One device, experiments as children, runs stored under each experiment

📦 Session
└─ 🔬 Device: MAPbI₃ n-i-p
   ├─ 🧪 J–V Sweep
   │  ├─ ▶ Run 11:02 (V_oc=1.08)
   │  └─ ▶ Run 11:15 (V_oc=1.09)
   ├─ 🧪 Impedance @ 0.9 V
   │  └─ ▶ Run 11:20
   └─ 🧪 Degradation 100 s
      └─ ▶ Run 11:30

Simplest mental model: one device open at a time, experiments hang off it, every Run button press saves a run you can re-open. Matches how a researcher actually works — one device under study at a time.

B

Workspace with multiple devices open at once, experiments per device

📦 Workspace
├─ 🔬 MAPbI₃ baseline
│  ├─ 🧪 J–V Sweep   ▶ Run 11:02
│  └─ 🧪 Impedance   ▶ Run 11:15
├─ 🔬 MAPbI₃ +Cs5%
│  └─ 🧪 J–V Sweep   ▶ Run 11:30
└─ 🔬 CIGS baseline
   └─ 🧪 J–V Sweep   ▶ Run 11:45

Several devices coexist in the tree so you can flip between them, run the same experiment on each, and compare. This is how Silvaco / Sentaurus projects actually look in practice.

C

Shallow buckets — Devices, Experiments, Results as three parallel folders

📦 Workspace
├─ 🔬 Devices
│  ├─ MAPbI₃ baseline
│  └─ CIGS baseline
├─ 🧪 Experiments
│  ├─ J–V @ 1 V/s
│  └─ Impedance sweep
└─ 📊 Results
   ├─ MAPbI₃ × J–V   (Run 11:02)
   └─ CIGS × J–V     (Run 11:15)

Devices and experiments are decoupled — you aim any experiment at any device. Results live in their own folder and cross-reference both. Most flexible for parameter studies but the highest learning curve.
