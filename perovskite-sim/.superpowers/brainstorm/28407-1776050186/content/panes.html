How should the workstation be divided into panes?

Three honest options. All keep the project tree on the left. The difference is how the centre and right-hand-side work.

Fixed 3-region layout
Tree
Device editor / parameter form
Main plot (J–V / Nyquist / ...)
Band diagram / density profile
Metric cards + run info
Solver console · Radau 247 steps · V_oc=1.08V · done in 12.3s

A — Fixed grid

Always the same regions in the same spots: tree left, editor top-centre, plot top-right, secondary view bottom-centre, metrics bottom-right, solver console as a strip along the bottom. Resizable splitters between them, but you can't tear a pane off or hide regions.

Feels like: early TCAD, SCAPS's results window, MATLAB's built-in layout.

Dockable panes with tabs
Tree
[Editor] [J–V Plot] [Band Diagram ×]
Active tab fills the area — drag a tab to split
Metric cards
Properties inspector
Solver console (collapsible)

B — Dockable / tabbed panes

Central area is a VS-Code-style tab/split region. Panes can be dragged to split horizontally or vertically, closed, or stacked as tabs. Bottom console and right-hand inspector are collapsible. Layout is saved per workspace.

Feels like: Silvaco DeckBuild, Sentaurus Workbench, VS Code, Lumerical.

Three-column focused
Tree
Device editor · experiment controls · metadata (one single scrollable column)
Results area — stacked plots (J–V + band diagram + density), click to expand any plot to full-width.
Solver console (always visible bottom strip)

C — Three-column focused

Tree · Config · Results. No secondary grid — the config column is one long scrollable form (device + experiment + metadata), and the results column stacks plots vertically with click-to-expand. Solver console is a thin always-visible strip at the bottom.

Feels like: Jupyter Lab, modern cloud-based simulators, Driftfusion Web. Pragmatic middle ground.
