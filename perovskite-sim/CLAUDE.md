## Project Overview

1D drift-diffusion + Poisson + mobile-ion simulator for thin-film solar cells (perovskite, CIGS, c-Si), with a FastAPI backend and a Vite/TypeScript/Plotly frontend that wraps the Python experiments as a browser UI.

```

‚îú‚îÄ‚îÄ backend/             FastAPI HTTP wrapper around perovskite_sim
```

Every layer talks to the next one through `DeviceStack` + `MaterialParams` (immutable frozen dataclasses); there is no shared mutable state.

## Simulator Architecture (`perovskite_sim/`)

**Core method.** Method of Lines: Scharfetter‚ÄìGummel finite elements on a tanh-clustered multilayer grid for space; `scipy.integrate.solve_ivp(Radau)` for time. Poisson is a tridiagonal solve with harmonic-mean face permittivity, evaluated once per RHS call (no inner Newton loop). State vector is `y = (n, p, P)` per grid node.

**MaterialArrays cache (hot path).** `solver/mol.py:build_material_arrays(x, stack)` returns an immutable `MaterialArrays` bundle holding every per-node / per-face array the RHS needs (`D_n_face`, `D_p_face`, `eps_r`, `N_A`, `N_D`, `P_ion0`, interface masks, boundary concentrations) plus a pre-factored Poisson operator. Every experiment builds this once per run and threads it through `assemble_rhs`, `run_transient`, `split_step`, `_compute_current`, and `_integrate_step` via the `mat=` kwarg. Rebuilding it per RHS call was the dominant cost before the cache landed ‚Äî do not reintroduce a code path that calls `build_material_arrays` inside the inner loop. Degradation is the one exception: it rebuilds `mat` only when the absorber-damage state actually changes (`mat_active` in `run_degradation`).

**Poisson LU cache.** `physics/poisson.py:factor_poisson(x, eps_r)` does one LAPACK `dgttrf` call and returns a `PoissonFactor` holding `(dl, d, du, du2, ipiv)`. `solve_poisson_prefactored(fac, rho, phi_left, phi_right)` then runs a single `dgttrs` per call ‚Äî ~40√ó faster than the legacy `solve_poisson` path. The factorization only depends on `(x, eps_r)`, which are constant across a transient, so it is cached inside `MaterialArrays.poisson_factor`. Legacy `solve_poisson` is kept as a fallback but is not on any hot path.

**Heterojunctions.** `chi` (electron affinity, eV) and `Eg` (band gap, eV) on each layer set the band offsets. Interface recombination uses per-interface `(v_n, v_p)` surface-recombination velocities carried by `DeviceStack.interfaces`.

**Band-offset contact BCs.** `DeviceStack.compute_V_bi()` derives the built-in potential from the Fermi-level difference across the heterostack (accounting for chi, Eg, doping, and ni). When chi=Eg=0 in all layers (legacy configs), it falls back to the manual `V_bi` field. The computed value is stored as `MaterialArrays.V_bi_eff` and used for voltage sweep range calculation (`V_max` defaults). The Poisson BC still uses `stack.V_bi` to match IonMonger's convention ‚Äî do not substitute `V_bi_eff` into the Poisson boundary without careful validation, as IonMonger treats V_bi as a free parameter representing the degenerate-doping limit.

**Thermionic emission (TE) at heterointerfaces.** At interfaces where the conduction-band offset |delta_Ec| or valence-band offset |delta_Ev| exceeds 0.05 eV, the SG flux is capped to the Richardson-Dushman thermionic emission limit (`fe_operators.thermionic_emission_flux`). This prevents the SG scheme from overestimating current across sharp band discontinuities (a known artifact when the band offset is resolved in a single grid spacing). The capping is applied in `continuity.py:carrier_continuity_rhs`. Richardson constants `A_star_n` and `A_star_p` default to the free-electron value (1.2017e6 A/(m¬≤¬∑K¬≤)) and can be overridden per layer in YAML configs. Interface faces where TE activates are pre-computed in `MaterialArrays.interface_faces`. Note: IonMonger does not use TE, so our V_oc is ~0.1 V higher than IonMonger's on the same parameter set; Phase 5 (tiered modes) will add a legacy mode that disables TE for exact IonMonger reproduction.

**Transfer-matrix optics (TMM).** `physics/optics.py` implements the coherent thin-film transfer-matrix method (Pettersson et al. 1999 / Burkhard et al. 2010) for position-resolved optical generation G(x). Each layer carries `optical_material` (string key for n,k CSV data in `perovskite_sim/data/nk/`) or `n_optical` (constant refractive index fallback). When any layer has `optical_material` set, `mol.py:_compute_tmm_generation` builds the TMM stack, loads the AM1.5G spectrum from `data/am15g.csv`, and computes G(x) once during `build_material_arrays`. The result is stored as `MaterialArrays.G_optical` and used in `assemble_rhs` instead of Beer-Lambert. Key physics: the absorption formula includes the `n_real / n_ambient` Poynting-vector correction so that R+T+A=1 (energy conservation). Without `optical_material`, the original Beer-Lambert `G = alpha * Phi * exp(-alpha*x)` path is unchanged. The `_inv2x2` batched matrix inverse in `optics.py` has a determinant guard (clamps `|det| < 1e-30` to avoid division by zero at wavelengths where the transfer matrix is singular); do not remove this guard.

**Activated presets:** `nip_MAPbI3_tmm.yaml`, `pin_MAPbI3_tmm.yaml` (Phase 2 ‚Äî Apr 2026). Both prepend a 1 mm `role: substrate` glass layer (optical-only) and set `optical_material` on every electrical layer. The vanilla `nip_MAPbI3.yaml` and `pin_MAPbI3.yaml` remain Beer-Lambert for back-compat with existing benchmarks. Substrate layers are filtered out of the drift-diffusion grid by `electrical_layers()` in `models/device.py`; the TMM spatial grid is offset by the substrate cumulative thickness so G(x) lands on the correct electrical nodes.

**Photon recycling / detailed-balance radiative recombination (Phase 3.1 ‚Äî Apr 2026).** When `SimulationMode.use_photon_recycling` is True and TMM optics are active, `build_material_arrays` scales the absorber `B_rad` by the Yablonovitch single-pass escape probability `P_esc = min(1, 1/(4¬∑n¬≤¬∑Œ±¬∑d))`, where `Œ±(Œª_gap) = 4œÄk(Œª_gap)/Œª_gap` is the intrinsic absorption at the band edge and `n` is the real index at the same wavelength (both read from the TMM layer's n,k arrays at `Œª_gap = hc/Eg`). The factor is applied per absorber layer using the role tag, so only layers marked `role: absorber` get rescaled ‚Äî transport layers are untouched. Implementation lives in `physics/photon_recycling.py` (`compute_p_esc`, `compute_p_esc_for_absorber`, `wavelength_at_gap`). The flag is on in both FAST and FULL, off in LEGACY. Comparing PR on vs off is only apples-to-apples when TMM is on in both runs (G(x) must be bit-identical); the regression test `tests/regression/test_photon_recycling_voc.py` does this by `monkeypatch`-ing `resolve_mode` on both `models.mode` and `solver.mol` before each V_oc measurement. The `radiative_limit.yaml` preset (absorber œÑ ‚Üí 1 s, Auger = 0, detailed-balance `B_rad = 4e-17`) is the canonical test vector and produces a ~78 mV V_oc boost with PR on, inside the literature window for MAPbI3. The reciprocal emission leg ‚Äî adding the reabsorbed fraction back into G(x) self-consistently ‚Äî landed as Phase 3.1b (see below); when `use_radiative_reabsorption` is on, the build-time `B_rad *= P_esc` scaling is skipped and the suppression is delivered per-RHS instead, which is bit-equivalent at V_oc but more accurate when n¬∑p is non-uniform.

**Self-consistent radiative reabsorption (Phase 3.1b ‚Äî Apr 2026).** Closes the photon-recycling loop opened in Phase 3.1. When `SimulationMode.use_radiative_reabsorption` is True (FULL only ‚Äî FAST keeps the cheaper Phase 3.1 build-time scaling), `build_material_arrays` does **not** apply the `B_rad *= P_esc` collapse; instead, for every absorber layer with `Eg > 0` it caches a `(mask, P_esc, thickness)` tuple on `MaterialArrays.absorber_masks / absorber_p_esc / absorber_thicknesses` and sets `has_radiative_reabsorption = True`. On every RHS call `assemble_rhs` then computes per absorber `R_tot = ‚à´ B_rad ¬∑ n ¬∑ p dx` over the cached mask and adds the reabsorbed fraction `G_rad = R_tot ¬∑ (1 ‚àí P_esc) / thickness` uniformly back onto the absorber's `G` nodes. Net effect: the bulk `B_rad` on the cache stays at its intrinsic value, full radiative emission acts as a sink in `carrier_continuity_rhs`, and the Yablonovitch-trapped fraction acts as a uniform source. In the spatially-uniform n¬∑p limit (which is exactly what `V_oc` looks like in the bulk because the quasi-Fermi splittings are flat away from the contacts) this reproduces the Phase 3.1 V_oc boost to within sub-mV ‚Äî the equivalence pinned by `tests/regression/test_radiative_reabsorption.py::test_radiative_reabsorption_matches_phase_3_1_at_voc`. Where 3.1b differs is the non-uniform regime: under strong injection, near the contacts, or in tandem sub-cells with engineered absorber profiles, emission near a high-`n¬∑p` region can feed generation in a low-`n¬∑p` region, redistributing carriers in a way the build-time scaling cannot capture. Like Phase 3.2 and 3.3 this hook intentionally breaks the "G computed once at build time" invariant ‚Äî but only on absorber nodes where `has_radiative_reabsorption` is True, so the cost is bounded. `assemble_rhs` always copies `G` before adding the source so `mat.G_optical` stays read-only across RHS calls. The flag is gated under `use_photon_recycling`: turning radiative reabsorption on without photon recycling is a no-op (no `P_esc` was ever computed). The backend label appends `¬∑ PR reabsorption` when active, distinguishing FULL from FAST in the SSE result payload (`backend/main.py:_describe_active_physics`).

**Field-dependent mobility (Phase 3.2 ‚Äî Apr 2026).** When `SimulationMode.use_field_dependent_mobility` is True and at least one layer sets a nonzero `v_sat_{n,p}` or `pf_gamma_{n,p}`, `assemble_rhs` recomputes effective per-face diffusion coefficients on every RHS call from the Poisson-solved face field `E_face = -‚àÇœÜ/‚àÇx`. Two empirical models compose multiplicatively: Poole-Frenkel `Œº_PF(E) = Œº‚ÇÄ ¬∑ exp(Œ≥_PF ¬∑ ‚àö|E|)` (field-assisted hopping in disordered / organic HTLs ‚Äî Œ≥ ~ 3e-4 (V/m)^-0.5 typical for spiro-OMeTAD) followed by Caughey-Thomas `Œº_CT(E) = Œº ¬∑ (1 + (Œº¬∑|E|/v_sat)^Œ≤)^(-1/Œ≤)` (velocity-saturation cap ‚Äî Œ≤ = 2 is the Canali form for silicon electrons, Œ≤ = 1 is the Thornber form for silicon holes; v_sat ~ 1e5 m/s is perovskite-realistic). Both primitives live in `physics/field_mobility.py` (`caughey_thomas`, `poole_frenkel`, `apply_field_mobility`). **Critically, this is the first physics path that intentionally breaks the "build once, reuse" invariant of `MaterialArrays`** ‚Äî Œº(E) depends on state, so `D_n_face` / `D_p_face` are recomputed per RHS call rather than cached. The hook is opt-in per layer: v_sat = 0 and Œ≥_PF = 0 (the defaults) leave Œº untouched at that face, and `MaterialArrays.has_field_mobility = False` when no layer opts in ‚Äî in which case the hot path is skipped entirely and results are bit-identical to Phase 3.1. `jv_sweep.compute_current_components` applies the same per-face correction to its terminal J_n / J_p calculation so charge conservation at the contacts is preserved. The integration test `tests/integration/test_field_mobility_integration.py` exercises the ON path (default presets leave it OFF; injecting v_sat perturbs the full J-V curve measurably without any NaN/Inf).

**Tiered simulation modes (Phase 5 ‚Äî Apr 2026).** Every Phase 1‚Äì3 physics upgrade is gated on a flag in `SimulationMode` (defined in `perovskite_sim/models/mode.py`). The three named tiers ‚Äî `LEGACY`, `FAST`, `FULL` ‚Äî bundle the nine flags into useful combinations so that callers can pick a fidelity / speed trade-off without editing individual flags. The tier acts as a **ceiling**: turning a flag on only enables the corresponding hook if the config actually provides the parameter the hook needs (TMM needs `optical_material`, TE needs non-zero `chi` and `Eg` with a real interface offset, selective contacts need at least one `S_*`, field-mobility needs `v_sat_{n,p}` or `pf_gamma_{n,p}`, radiative reabsorption needs `use_photon_recycling` plus an absorber with `Eg > 0` under TMM, etc.). This "enable-if-configured" pattern means the tier strings are safe to set on every preset ‚Äî flags are silent when inactive rather than forcing physics that the stack cannot support. Flag matrix: **LEGACY** turns everything off for IonMonger reproducibility. **FAST** turns on every *build-once* upgrade (TE, TMM, dual ions, trap profile, T-scaling, photon recycling) but keeps the three *per-RHS* upgrades off ‚Äî radiative reabsorption (3.1b), field-dependent mobility (3.2), and selective contacts (3.3) all break the MaterialArrays build-once invariant, so FAST tier skips them. **FULL** turns everything on, including the three per-RHS hooks. A YAML preset can pin a tier with a top-level `device.mode: legacy|fast|full`; `DeviceStack.mode` defaults to `"full"` so any config that does not pin a mode gets the full physics set. `resolve_mode` (in `models/mode.py`) is idempotent on `SimulationMode` instances, parses strings case-insensitively, and returns `FULL` for `None`. The backend renders the resolved tier into the SSE result payload via `_describe_active_physics`, which produces strings like `"FULL  band offsets ¬∑ TE ¬∑ TMM ¬∑ dual ions ¬∑ trap profile ¬∑ T-scaling ¬∑ photon recycling ¬∑ PR reabsorption ¬∑ Œº(E) ¬∑ Robin contacts"` ‚Äî the per-RHS fragments only appear when the corresponding flag is on, so FAST and LEGACY get a shorter label. The tier system is invariant-tested in `tests/unit/models/test_mode.py` (flag monotonicity: `LEGACY ‚äÜ FAST ‚äÜ FULL`), and end-to-end in `tests/regression/test_tier_regression.py` (LEGACY reproduces FAST/FULL on a plain preset; FAST ignores per-RHS opt-ins even when the config provides them; LEGACY disables TE even with band offsets configured; YAML `mode:` round-trips through the full loader).

**Selective / Schottky outer contacts (Phase 3.3 ‚Äî Apr 2026).** The pre-3.3 outer boundary treatment was a hard Dirichlet pin: at nodes 0 and N-1 the carrier densities were clamped to their ohmic-equilibrium values (`mat.n_L`, `mat.n_R`, `mat.p_L`, `mat.p_R`) and `dn` / `dp` at those indices were zeroed so the solver never touched them. This is the correct limit for an ideal ohmic contact but cannot represent a selective (electron- or hole-blocking) or Schottky-barrier contact, which are the dominant outer-contact types in real perovskite devices. When `SimulationMode.use_selective_contacts` is True and `DeviceStack` supplies at least one of `S_n_left`, `S_p_left`, `S_n_right`, `S_p_right` (in m/s), `build_material_arrays` sets `has_selective_contacts = True` and the four `S` values propagate onto `MaterialArrays`. `assemble_rhs` then (a) keeps the boundary density for that carrier/side from the integrated state vector rather than overriding it, (b) computes a Robin-type flux `J = ¬±q ¬∑ S ¬∑ (density ‚àí density_eq)` via `physics/contacts.selective_contact_flux`, and (c) plumbs that flux through `carrier_continuity_rhs` as an extra `J_n_L` / `J_n_R` / `J_p_L` / `J_p_R` entry in `params`, which replaces the zero-flux pad at that boundary and skips the `dn[0]=0` / `dn[-1]=0` pin for that specific carrier. Sides left as `None` (the default, and what every shipped preset supplies) keep the exact pre-3.3 Dirichlet behaviour, so the feature is backward-compatible at the numerics level. `S ‚Üí ‚àû` recovers the ohmic limit; `S = 0` is a perfectly blocking (Neumann) contact; finite S allows the boundary node to evolve with a Robin time constant `~ dx_cell / S`. YAML accepts both a nested `device.contacts: { left: {S_n, S_p}, right: {S_n, S_p} }` block (preferred) and flat top-level `device.S_n_left` etc. keys. `experiments/jv_sweep._state_fields`, `solver/mol.split_step`, and the sign-aware `dn[boundary] = 0` overwrites in `assemble_rhs` all respect the Robin/Dirichlet split per carrier / per side. Schottky helpers `schottky_equilibrium_n(N_c, œÜ_B, V_T)` and `schottky_equilibrium_p(N_v, œÜ_B, V_T)` live alongside the Robin primitives in `physics/contacts.py` ‚Äî these compute the thermionic equilibrium density at a barrier and are intended as the source of the `n_eq` argument to `selective_contact_flux` when the user wants a Schottky rather than ohmic-equilibrium reference. The `configs/selective_contacts_demo.yaml` preset exercises the full path on the nip MAPbI3 stack (small wrong-sign S, moderate matched-carrier S) and the integration test `tests/integration/test_selective_contacts_integration.py` covers the activation flag, YAML round-trip, finite-RHS under aggressive Robin fluxes including the `S=0` Neumann limit, and a J-V smoke test that fails if the Robin hook is silently bypassed.

**Temperature-dependent radiative rate and bandgap (Phase 4b ‚Äî Apr 2026).** Extends the Phase 4 temperature-scaling hook (`SimulationMode.use_temperature_scaling`, gated via FAST/FULL, off in LEGACY) to cover two more parameters that matter at non-300 K: the band-to-band radiative coefficient `B_rad` and the optical bandgap `Eg`. Both are opt-in via new `MaterialParams` fields so pre-Phase-4b configs are bit-identical. `B_rad_at_T(B_300, T, gamma)` implements the empirical power law `B(T) = B_300 ¬∑ (T/300)^Œ≥` with `Œ≥ = 0` as the opt-out default (literature typical for MAPbI3 is `Œ≥ ‚âà -1.5`, the Varshni-free detailed-balance scaling for a non-degenerate semiconductor at fixed bandgap); setting `MaterialParams.B_rad_T_gamma` to a non-zero value activates the scaling in `build_material_arrays` on every per-layer `B_rad` write. `eg_at_T(Eg_300, T, alpha, beta)` implements the Varshni bandgap shift **referenced to 300 K** (rather than the classical Varshni-to-0-K form), so `MaterialParams.Eg` continues to mean "the bandgap at 300 K" and the shift is applied only when `varshni_alpha ‚âÝ 0`: `Eg(T) = Eg_300 ‚àí Œ±¬∑[T¬≤/(T+Œ≤) ‚àí T_REF¬≤/(T_REF+Œ≤)]`. Silicon uses `Œ± ‚âà 4.73e-4 eV/K, Œ≤ ‚âà 636 K` (heating narrows Eg); MAPbI3 is opposite ‚Äî Eg *widens* with heating, reproduced by `Œ± ‚âà ‚àí3e-4 eV/K, Œ≤ ‚âà +200 K`. The Varshni-shifted `Eg_T` is fed **both** to `chi[mask]`/`Eg[mask]` (so downstream thermionic emission, band offsets, and photon recycling see the shifted band edges) **and** to `ni_at_T` (so `ni¬≤(T) ‚àù exp(‚àíEg(T)/kT)` is self-consistent with the shifted bandgap). The degenerate case `T + Œ≤ = 0` returns `Eg_300` rather than dividing by zero. Both hooks respect the LEGACY gate ‚Äî if `sim_mode.use_temperature_scaling` is off, `B_rad` and `Eg` stay at their 300 K values even when the new fields are populated (pinned by the `test_legacy_mode_*` tests in `tests/unit/solver/test_temperature_scaling_plumbing.py`). The full test surface is 14 unit tests in `tests/unit/physics/test_temperature.py` (identity-at-300 K, power-law exponent, zero-input sentinels, default-gamma-is-zero, Varshni sign checks for silicon vs MAPbI3 parameter sets, degenerate-denominator guard) plus 7 plumbing tests in `tests/unit/solver/test_temperature_scaling_plumbing.py` (default leaves arrays unchanged, opt-in scales correctly, legacy-mode freezes at 300 K, shipped configs remain bit-identical to the pre-Phase-4b build path). Phase 4b does not add a new tier flag ‚Äî it reuses the existing `use_temperature_scaling` gate, which keeps LEGACY bit-identical to IonMonger even when a preset carelessly sets `B_rad_T_gamma` or `varshni_alpha`.

**Position-dependent trap profiles (Phase 4a ‚Äî Apr 2026).** The bulk SRH lifetime on each layer represents the cleanest part of the film; in real perovskite devices grain-boundary defects concentrate near each transport-layer interface and the local SRH lifetime drops accordingly. Phase 4a turns the original inline Phase 4 profile into a reusable physics module and adds a second shape for defect layers with a well-defined finite extent. The primitives live in `physics/traps.py`: `exponential_edge_profile(x_local, thickness, N_t_interface, N_t_bulk, L_d)` returns `N_t(x) = N_t_bulk + (N_t_interface ‚àí N_t_bulk) ¬∑ (e^{‚àíx/L_d} + e^{‚àí(d‚àíx)/L_d})` (Phase 4 baseline ‚Äî two exponentials added rather than maxed so a thin layer with overlapping decay tails smoothly saturates at `N_t_interface` instead of double-counting one side); `gaussian_edge_profile(...)` returns the same construction with `G(s) = exp(‚àí(s/sigma)¬≤)` and falls off faster in the tail (`~1e-4` at 3œÉ vs `~0.05` at 3¬∑L_d for the exponential) ‚Äî useful when the interface defect slab has a measured finite thickness rather than a long tail; `tau_from_trap_density(tau_bulk, N_t, N_t_bulk)` applies the SRH inverse-density rule `tau(x) = tau_bulk ¬∑ N_t_bulk / N_t(x)` with the denominator floored at `N_t_bulk` so that `N_t_interface < N_t_bulk` (a passivation profile) still yields a physical, monotone correction; `has_trap_profile_params(params)` is the presence check used by `build_material_arrays` to decide whether to call the profile. Shape selection is per-layer via the new `MaterialParams.trap_profile_shape` field (`"exponential"` default | `"gaussian"`), wired through `config_loader.py` as `layer_cfg.get("trap_profile_shape", "exponential")`. `L_d = 0` or `sigma = 0` are degenerate "no profile" sentinels that return `N_t_bulk` everywhere, and `L_d = 0` on a layer that still opts in simply leaves `tau` at its layer-uniform bulk value. `build_material_arrays` also caches the per-node trap density on `MaterialArrays.N_t_node` (zeros on nodes that belong to a layer with no trap params, `N_t_bulk` on layers that only specify the bulk density without a profile, and the full `N_t(x)` on layers that opt in), plus a boolean `has_trap_profile` that reports whether at least one layer triggered the profile. These diagnostics let downstream hooks (interface SRH, band-tail absorption, mobility degradation) reuse the same `N_t(x)` without recomputing the profile. The whole module is gated by `SimulationMode.use_trap_profile` (on in FAST and FULL, off in LEGACY); when the flag is off `build_material_arrays` skips the profile entirely and `N_t_node` reflects the uniform-bulk fallback. Unit tests in `tests/unit/physics/test_traps.py` pin the endpoint formula, midpoint bulk recovery, monotonic decay into the interior, Gaussian tail decays faster than exponential at ~3¬∑L, zero-length sentinel yields flat bulk, `tau_from_trap_density` passivation floor, and the `has_trap_profile_params` presence check. The existing integration test `tests/integration/test_temperature_trap.py` continues to cover the end-to-end `build_material_arrays` path (now through the refactored module) without modification.

**Experiments** (`perovskite_sim/experiments/`):
- `jv_sweep.run_jv_sweep` ‚Äî forward then reverse scan; reuses the previous steady state as initial condition so ionic memory is preserved and the hysteresis loop comes out of the physics, not post-processing. A per-step `_JV_RADAU_MAX_NFEV = 100_000` guard aborts any single voltage step that consumes too many RHS evaluations (prevents the solver from hanging on pathological substrate-stack configs). Supports `illuminated=False` for dark J-V (G=0, dark-equilibrium start) and `save_snapshots=True` for collecting `SpatialSnapshot` at every voltage point.
- `jv_sweep.compute_current_components` ‚Äî decomposes the terminal current into J_n (electron), J_p (hole), J_ion (ionic, both species), and J_disp (displacement) at every mesh face. Uses the same SG flux formulas as `assemble_rhs` but multiplied by Q for A/m¬≤ units. Ion current includes the steric Blakemore correction and dual-ion support.
- `jv_sweep.extract_spatial_snapshot` ‚Äî extracts a `SpatialSnapshot` (phi, E, n, p, P, rho) from a packed state vector at a given voltage. Used internally by `save_snapshots` and available as a standalone API.
- `impedance.run_impedance` ‚Äî at each frequency integrates a few AC cycles and extracts amplitude/phase with a lock-in (sin/cos multiply + low-pass). Adds the displacement current `Œµ‚ÇÄ¬∑Œµ_r¬∑‚àÇE/‚àÇt`.
- `degradation.run_degradation` ‚Äî long-time transient; at each probe time it takes a **frozen-ion snapshot**: a `replace`-d copy of the stack with `D_ion = 0` in every layer is used for a short settle integration at each probe voltage (`_freeze_ions` + `_measure_snapshot_metrics`). This measures the instantaneous electronic response under the current ionic configuration and is the only correct way to compute snapshot J‚ÄìV without averaging over ion drift.
- `tpv.run_tpv` ‚Äî transient photovoltage experiment. Equilibrates at V_oc under illumination, applies a fractional generation pulse (delta_G_frac), tracks J(t) at fixed V_app=V_oc, converts to V(t) via the small-signal relation delta_V = -J * R_oc, and fits a mono-exponential decay time tau. Returns a `TPVResult` with t, V, J arrays plus V_oc, tau, delta_V0.
- `tandem_jv.run_tandem_jv` ‚Äî 2T monolithic tandem driver. Runs a combined TMM over the full stack (`physics/tandem_optics.py`) to partition G(x) into top/bottom sub-cell profiles, then performs independent drift-diffusion J‚ÄìV sweeps with `fixed_generation`, and series-matches at a common current grid (`series_match_jv`). Returns a `TandemJVResult` holding per-sub-cell results plus the tandem J‚ÄìV curve.

**Tandem optics** (`physics/tandem_optics.py`): Runs one TMM over the entire tandem stack (top + junction + bottom), then splits absorption by layer. Junction layers count as parasitic absorption. The `TandemGeneration` dataclass holds `G_top(x)` and `G_bot(x)` arrays. The tandem config model lives in `models/tandem_config.py` (`TandemConfig`, `JunctionLayer`).

**2D extension ‚Äî Stage A lateral-uniform parity (Phase 6 ‚Äî Apr 2026).** Lives in `perovskite_sim/twod/` as a parallel sub-package ‚Äî it does not touch the 1D solver hot path. The 2D state is `(n(y, x), p(y, x))` only; ions are held as a static Poisson background (`P_ion_static_1d`) extruded from the 1D illuminated steady state, so Stage A has no ionic memory across voltage steps. Grid: `build_grid_2d` produces a tensor-product grid with the existing tanh-clustered y-axis (one `Layer(thickness, Ny_per_layer)` per electrical layer) and a uniform x-axis of `Nx + 1` nodes spanning `lateral_length`. Poisson: `physics/poisson_2d.py` assembles a sparse 5-point stencil with harmonic-mean face permittivity and `scipy.sparse.linalg.splu` factor cached on `MaterialArrays2D` ‚Äî the same Dirichlet contacts as 1D (`phi[0,:]=0`, `phi[-1,:]=V_bi - V_app`) and lateral periodic or Neumann BCs. The Poisson RHS includes the frozen-ion term `Q*(P_ion_static - P_ion0_2d)`, which is critical for matching 1D's $\rho$ ‚Äî without it the 2D $\varphi$ diverges at heterointerfaces and SG fluxes blow up. Continuity: `twod/flux_2d.py` implements vectorised 2D Scharfetter-Gummel for both directions with **harmonic-mean face-D** averaging (matches 1D `MaterialArrays.D_n_face`); the SG flux uses `phi_n = phi + chi`, `phi_p = phi + chi + Eg` so band offsets are preserved at heterointerfaces. Thermionic-emission capping: `twod/continuity_2d.py` caps `Jy_n` / `Jy_p` at vertical heterointerfaces (where `|delta_Ec| > 0.05` eV or `|delta_Ev| > 0.05` eV) using the same Richardson-Dushman bound as 1D `continuity.py`. The `dn = +(div_x_n + div_y_n)/Q + G - R` sign for electrons (charge $-q$) and `dp = -(div_x_p + div_y_p)/Q + G - R` for holes are pinned by `tests/regression/test_twod_validation.py` ‚Äî getting either sign wrong produces immediate Radau failure at V=0.

The Stage-A J-V driver `twod/experiments/jv_sweep_2d.run_jv_sweep_2d` bootstraps via 1D `solve_illuminated_ss` (or `solve_equilibrium` for dark) on the same y-grid, freezes the equilibrated ion profile via `P_ion_static_1d`, broadcasts $(n, p)$ to 2D, and walks the voltage grid with a per-step `run_transient_2d` settle (Radau-LU on the ~250√ó250 system at the test-grid sizes). The validation gate `tests/regression/test_twod_validation.py` runs the 1D and 2D solvers on the Beer-Lambert MAPbI3 preset with frozen ions and matched grids (1D `N_grid=31`, 2D `Ny_per_layer=10` √ó 3 electrical layers + 1 = 31) and asserts: $|V_{oc}^{2D} - V_{oc}^{1D}| \le 10^{-4}$ V, $|\Delta J_{sc}|/|J_{sc}| \le 5\times10^{-4}$, $|\Delta \text{FF}| \le 10^{-3}$, lateral invariance of $n$ at $V_{oc}$ to $\le 10^{-9}$, divergence of total electron+hole current bounded at interior nodes, and Poisson Dirichlet BCs match exactly. Final passing values: bit-identical 1D and 2D (`V_oc=910.634` mV, `J_sc=424.036` A/m¬≤, FF=0.8247).

Backend dispatch: `kind="jv_2d"` in `backend/main.py` accepts `lateral_length`, `Nx`, `Ny_per_layer`, `V_max`, `V_step`, `settle_t`, `illuminated`, `lateral_bc`, `save_snapshots` and serialises `JV2DResult` (V, J, grid_x in nm, grid_y in nm, lateral_bc, snapshots). The frontend pane `frontend/src/workstation/panes/jv-2d-pane.ts` is registered under the `'2D / Microstructural (Stage A)'` group as `J‚ÄìV Sweep (2D)`; the result renderer in `main-plot-pane.ts` flips the J sign to match the 1D solar convention. The TMM preset `nip_MAPbI3_tmm` and the BL preset `nip_MAPbI3` (and the alias `configs/twod/nip_MAPbI3_uniform.yaml`) are the supported Stage-A presets ‚Äî the Stage-A regression uses BL because the TMM preset has the pre-existing 1D coupled-solver regression at V=0.21 V (see `project_tmm_jv_regression_021.md`).

Stage-A scope boundaries ‚Äî Phase 6 deliberately does **not** include 2D ions, 2D radiative reabsorption / field mobility / Robin contacts, or per-grain $\tau$ profiles. Those reuse the same `MaterialArrays2D` cache slots (which already hold `chi`, `Eg`, `A_star_n/p`, `interface_y_faces`, etc.) and the same `assemble_rhs_2d` skeleton, but are out of scope for the Stage-A validation gate.

**2D Microstructure ‚Äî Stage B (Phase 6 ‚Äî Apr 2026).** Lateral grain boundaries with reduced SRH lifetimes, painted onto the absorber via `perovskite_sim/twod/microstructure.py`. The data model is two frozen dataclasses: `GrainBoundary(x_position, width, tau_n, tau_p, layer_role)` represents one vertical band, and `Microstructure(grain_boundaries=...)` is the container. `build_tau_field(grid, ustruct, tau_n_bulk_per_y, tau_p_bulk_per_y, layer_role_per_y)` extrudes the bulk $\tau$ along $x$ and then overrides cells inside each GB band where `|x - gb.x_position| < gb.width/2` AND the role tag at that $y$ matches `gb.layer_role`.     - x_position: 250e-9   # m, GB centre
```yaml
      width:      5e-9     # m, GB band width
      tau_n:      5e-8     # s, electron SRH lifetime inside GB

```

```
DeviceStack.S_p_left  ‚Üí MaterialArrays2D.S_p_top
DeviceStack.S_n_right ‚Üí MaterialArrays2D.S_n_bot  (y=Ny-1, ETL / n-contact)
DeviceStack.S_p_right ‚Üí MaterialArrays2D.S_p_bot
```

The sign table (verified from first principles; `dp` signs are opposite to `dn` because `dp = ‚àídiv_p/Q`):

```
dn[0,  :] -= J_n_top / (Q * hy_top)   # top electrons: J_n_top = +q¬∑S¬∑(n‚àín_eq)
dp[0,  :] += J_p_top / (Q * hy_top)   # top holes:     J_p_top = ‚àíq¬∑S¬∑(p‚àíp_eq)  ‚Üê +=
dn[-1, :] += J_n_bot / (Q * hy_bot)   # bot electrons: J_n_bot = ‚àíq¬∑S¬∑(n‚àín_eq)
dp[-1, :] -= J_p_bot / (Q * hy_bot)   # bot holes:     J_p_bot = +q¬∑S¬∑(p‚àíp_eq)  ‚Üê -=
```

Equilibrium densities use `mat.n_eq_left` / `mat.p_eq_left` for y=0 and `mat.n_eq_right` / `mat.p_eq_right` for y=Ny-1 (existing `(Nx,)` arrays; note the code comments on these fields had "top/bottom" reversed ‚Äî fixed in this stage). **2D self-consistent radiative reabsorption ‚Äî Stage B(c.3) (Phase 6 ‚Äî Apr 2026).** Ports the 1D Phase 3.1b reabsorption hook to `assemble_rhs_2d`. When `SimulationMode.use_radiative_reabsorption` is True AND `SimulationMode.use_photon_recycling` is True AND `MaterialArrays.has_radiative_reabsorption` is True (the 1D side has populated per-absorber tuples under TMM optics), `build_material_arrays_2d` translates `mat1d.absorber_masks` to 2D `absorber_y_ranges_2d` (each is a contiguous `(y_lo, y_hi)` half-open range derived from `layer_role_per_y == "absorber"`) and pre-computes `absorber_areas_2d = thickness √ó lateral_length` per absorber.

On every RHS call, `assemble_rhs_2d` calls `recompute_g_with_rad_2d` (in `perovskite_sim/twod/radiative_reabsorption_2d.py`) which: (1) for each absorber, computes `emission = B_rad[y_lo:y_hi, :] ¬∑ n[y_lo:y_hi, :] ¬∑ p[y_lo:y_hi, :]` of shape `(n_y_abs, Nx)`; (2) trapezoid in y first (axis=0) ‚Üí `(Nx,)`; then trapezoid in x ‚Üí scalar `R_tot_2D`; (3) skips if `R_tot_2D ‚â§ 0`, `area ‚â§ 0`, `P_esc ‚â• 1`, or fewer than 2 nodes on either axis; (4) computes `G_rad = R_tot_2D ¬∑ (1 ‚àí P_esc) / area`; (5) augments `G[y_lo:y_hi, :] += G_rad` (uniform redistribution over absorber 2D area). The cached `mat.G_optical` is **never mutated** ‚Äî the helper returns a NEW `(Ny, Nx)` array. The augmented `G_to_use` is then forwarded to `continuity_rhs_2d`. When the flag is False, `G_to_use is mat.G_optical` (same Python object) ‚Äî bit-identical to Stage A / B(a) / B(c.1) / B(c.2).

**Bit-equivalent to 1D in the lateral-uniform limit.** With constant `dx`, the lateral trapezoid evaluates to `lateral_length √ó emission_y(y)`, and dividing by `area = thickness √ó lateral_length` recovers `emission_y / thickness` ‚Äî exactly 1D's `G_rad = R_tot ¬∑ (1 ‚àí P_esc) / thickness`. The 2D y-range derives from `layer_role_per_y == "absorber"` (one half-cell narrower than the 1D inclusive mask at the HTL/absorber interface, which uses upstream-tagging in 2D); this trades ~1.3 mV V_oc against the 5 mV B(c.3) parity envelope and keeps the absorber row semantics consistent with Stage B(a) microstructure painting.

On `RuntimeError` from `run_transient_2d` AND `mat.has_radiative_reabsorption_2d` AND `illuminated`, `_bake_radiative_reabsorption_step_2d` evaluates `R_tot_2D` once at the entry state of the failed voltage step, folds `G_rad` into a step-local `G_optical` copy, clears the flag and zeros all four absorber tuples via `dataclasses.replace`, and retries `run_transient_2d` once with the baked `mat`. Mirrors 1D `experiments/jv_sweep.py:_bake_radiative_reabsorption_step` exactly.

### Solver gotcha ‚Äî RHS finite-check

`assemble_rhs` includes a `np.all(np.isfinite(dydt))` guard that raises `ValueError` if any element is NaN or Inf. This catches blow-ups from singular TMM matrices, zero-thickness layers, or extreme doping imbalances early ‚Äî before Radau can silently accept them and produce garbage. Do not remove this check.

### Solver gotcha ‚Äî Radau max_step cap

Near flat-band (V ‚âà V_bi) the Jacobian is nearly singular and Radau's adaptive LTE estimator under-reports the error, letting it accept one giant step on the wrong branch of the implicit system. This manifests as an unphysical J‚ÄìV spike (e.g. `J=258` sandwiched between `188` and `101`). **All three experiments therefore cap `max_step = Œît / k`** on every `run_transient` sub-interval (`jv_sweep._integrate_step`, `impedance`, `degradation._measure_snapshot_metrics`). Do not remove these caps; if you need tighter control, lower the divisor instead. Any new experiment that calls `run_transient` across a voltage where the Jacobian can become near-singular must apply the same cap.

## Backend (`backend/main.py`)

Thin FastAPI wrapper. Two endpoint families:

**Legacy blocking endpoints** ‚Äî `POST /api/jv`, `/api/impedance`, `/api/degradation`. Run synchronously and return the full result in one response. Kept for backwards compatibility; the frontend no longer uses them.

**Streaming job endpoints** (frontend uses these):
- `POST /api/jobs` ‚Äî body `{kind, config_path|device, params}`. Dispatches the experiment onto a worker thread via `JobRegistry.submit` and returns `{job_id}`. The closure captured in `_run(reporter)` passes `reporter.report` as the `progress=` kwarg to the experiment.
- `GET /api/jobs/{id}/events` ‚Äî Server-Sent-Events stream. Named events are `progress` (JSON `{stage, current, total, eta_s, message}`), `result` (final serialized result), `error` (on exception), and `done` (always last). The handler uses `run_in_executor` + a 0.5 s drain timeout to avoid blocking the event loop, and emits `: keepalive` SSE comments in the gap between progress frames.

`backend/progress.py` and `backend/jobs.py` are the pub/sub primitive and the thread-per-job registry behind this. Any new experiment that wants progress should take a `progress: Callable[[str, int, int, str], None] | None = None` kwarg (see `experiments/jv_sweep.py` for the pattern) and the backend `_run` closure wraps it with `reporter.report`.

`GET /api/configs` auto-scans `configs/`, so dropping a new YAML in that folder makes it visible to the frontend with no code change.

### Development loop ‚Äî restart uvicorn after backend edits

`uvicorn backend.main:app --reload` picks up most edits, but the watcher is set on the SolarLab root and occasionally misses changes under `perovskite-sim/backend/`. If `/api/jobs` or a new endpoint returns 404 after an edit, kill and restart the process rather than trusting `--reload`.

### YAML scientific-notation gotcha

PyYAML's default loader is YAML 1.1, which only treats `1.0e-9` as a float ‚Äî a bare `1e-9` (no decimal point) is returned as a **string**. The frontend's numeric editor then throws `v.toExponential is not a function` when rendering such a field. The fix lives in `backend/main.py:_coerce_numbers`, a recursive pass applied in `get_config` that turns any numeric-looking string into a float. When editing the backend do not bypass this coercion, and when adding new YAML configs you can use either form, but `1e-9` is fine because the backend normalises it.

Plain TypeScript (no framework). `main.ts` wires five tabs ‚Äî `J‚ÄìV Sweep`, `Impedance`, `Degradation`, `Tutorial`, `Algorithm` ‚Äî each mounted lazily into its panel `<section>`. Key modules:

- `api.ts` ‚Äî typed fetch wrappers for the backend

### Backend URL: absolute, not proxied

`api.ts` and `job-stream.ts` both hardcode `http://127.0.0.1:8000` as the base URL. The Vite dev-server proxy is configured for `/api` in `vite.config.ts`, but its SPA history fallback intercepts `/api/configs` before the proxy has a chance to match, returning `index.html` ‚Äî the JSON parse then fails with `Unexpected token '<'`. Hitting the backend directly sidesteps this. CORS is already wide open in `backend/main.py`, so this works from both `npm run dev` and `npm run preview`/`dist`.

SSE notes worth knowing:
- `EventSource` fires a native `error` event when the server closes the connection. `job-stream.ts:streamJobEvents` ignores events whose `e.data` is falsy ‚Äî only SSE `event: error` frames sent from the backend are surfaced to the user.
- Progress frames and the terminal `done` frame go through named SSE events (`event: progress`, `event: result`, `event: error`, `event: done`), not the default message channel, so each has its own `addEventListener`.

### Streaming-job + progress-bar pattern

Each experiment panel (`panels/{jv,impedance,degradation}.ts`) follows the same shape:
1. Instantiate a `ProgressBarHandle` once in `mount*Panel` via `createProgressBar(progressContainer)`.
2. On button click, call `startJob(kind, device, params)` ‚Üí get `job_id`.
3. Open the stream with `streamJobEvents<TResult>(jobId, { onProgress, onResult, onError, onDone })`.
4. `onResult` calls the panel-local `render*Results`; `onDone` re-enables the Run button.

The button is disabled between `startJob` and `onDone`, and the progress bar is `reset()` on every run so a second click doesn't briefly show the previous run's "done" state.

## Configs (`configs/`)

Presets shipped with the repo:

- **Beer-Lambert:** `nip_MAPbI3`, `pin_MAPbI3`, `ionmonger_benchmark` (Courtier 2019), `driftfusion_benchmark`, `cigs_baseline` (ZnO/CdS/CIGS, `D_ion=0`), `cSi_homojunction` (n+/p wafer, `D_ion=0`)
- **TMM-enabled:** `nip_MAPbI3_tmm`, `pin_MAPbI3_tmm`, `ionmonger_benchmark_tmm`, `driftfusion_benchmark_tmm`
- **Tandem sub-cells:** `nip_wideGap_FACs_1p77` (1.77 eV top), `nip_SnPb_1p22` (1.22 eV bottom)
- **Tandem config:** `tandem_lin2019` (2T monolithic, uses `TandemConfig` not `DeviceStack`)

The YAML schema mirrors `MaterialParams` + `DeviceStack.interfaces`; see any existing file for the field list. Non-perovskite stacks must set `D_ion = 0` in every layer ‚Äî the ion equations still integrate but contribute zero flux. Tandem configs use a separate `TandemConfig` schema (`models/tandem_config.py`) that references two sub-cell configs plus junction layers.

Practical solver envelope: the Radau transient handles sub-micron perovskite stacks comfortably (ionmonger reference ‚âà 25 s for a full J‚ÄìV). Thick inorganic absorbers (2 ¬µm CIGS, 180 ¬µm Si) are structurally valid and `solve_equilibrium` converges, but a full transient J‚ÄìV sweep is impractical at default tolerances. Use thinner absorbers or a coarser grid when iterating, and prefer equilibrium-level tests for those configs.

## Data Model Invariants

- `MaterialParams`, `SolverConfig`, `LayerSpec`, `DeviceStack` are **frozen dataclasses**. Never mutate in place ‚Äî use `dataclasses.replace(...)` to produce a new instance (see `degradation._freeze_ions` for the canonical pattern).
- All physical quantities are SI (m, s, V, A/m¬≤, m‚Åª¬≥). The only exceptions are `chi` and `Eg` which are in eV (numerically equal to volts through `q`).
- `DeviceStack.interfaces` is a tuple of `(v_n, v_p)` pairs with length `len(layers) - 1`, ordered from contact side to contact side.

## Test Structure

```
tests/
‚îú‚îÄ‚îÄ unit/          per-module physics + solver + experiments tests
‚îú‚îÄ‚îÄ integration/   end-to-end experiment runs on preset configs
‚îî‚îÄ‚îÄ regression/    physical sanity checks (V_oc range, J_sc bounds, ...)
```

`pytest` defaults (from `pyproject.toml`) exclude `-m slow`. Coverage is **opt-in** ‚Äî pass `--cov=perovskite_sim --cov-report=term-missing` explicitly when you want a report. The regression suite is where to add new "result should look physically reasonable" checks.

Regression tests:
- `test_jv_regression.py` ‚Äî V_oc / J_sc / FF / HI envelopes for all presets
- `test_tmm_baseline.py` ‚Äî TMM-specific baselines (slow, BLAS-pinned)
- `test_conservation.py` ‚Äî energy conservation (R+T+A=1) and J-V monotonicity checks; catches TMM numerical issues and solver blow-ups

### Test gotcha ‚Äî slow suite BLAS thread pinning

The TMM regression suite (`tests/regression/test_tmm_baseline.py`) drives ~4700 calls to `scipy.linalg.lu_factor` on the ~300√ó300 Radau Jacobian. These matrices are too small for multi-threaded BLAS to pay off ‚Äî on a 10-core box OpenBLAS spins up every LU call across all cores and the thread-creation + contention overhead turns a 14 s test into a 5-10 minute test. A Phase 2a investigation wasted four run-kills diagnosing this as a "stall" (runs were being terminated at ~4 min wall, but they would have taken another 2-3 min each to finish under oversubscription). It is NOT a hang.

`tests/conftest.py` pins BLAS threads to 1 via `threadpoolctl` whenever the `slow` marker is selected (excluding the default `-m 'not slow'` unit run). numpy/scipy must be imported inside the hook before calling `threadpool_limits`, because threadpoolctl only sees already-loaded backends. Do not remove this hook; also do not re-bake `--cov=perovskite_sim` into `pyproject.toml` `addopts` ‚Äî pytest-cov's line tracer adds a further ~17√ó overhead on the same hot loop and is equally sticky (the tracer is installed during pytest-cov's own `pytest_configure`, before any user conftest gets a chance to stop it).

Canonical invocations:
- Unit + integration: `pytest` (default, ~15 s, no coverage)
- Unit with coverage: `pytest --cov=perovskite_sim --cov-report=term-missing`
- Slow regression: `pytest -m slow` (BLAS pinned automatically, ~27 s for the TMM baselines)
