## Project Overview

1D drift-diffusion + Poisson + mobile-ion simulator for thin-film solar cells (perovskite, CIGS, c-Si), with a FastAPI backend and a Vite/TypeScript/Plotly frontend that wraps the Python experiments as a browser UI.

## Repository Layout

```
perovskite-sim/
в”ңв”Җв”Җ perovskite_sim/      Python simulation library (installable package)
в”ңв”Җв”Җ backend/             FastAPI HTTP wrapper around perovskite_sim
в”ңв”Җв”Җ frontend/            Vite + TS + Plotly single-page UI
в”ңв”Җв”Җ configs/             YAML device presets consumed by both CLI and UI
в”ңв”Җв”Җ tests/               pytest suite (unit + integration + regression)
в””в”Җв”Җ notebooks/           exploratory notebooks / benchmarks
```

Every layer talks to the next one through `DeviceStack` + `MaterialParams` (immutable frozen dataclasses); there is no shared mutable state.

## Simulator Architecture (`perovskite_sim/`)

**Core method.** Method of Lines: ScharfetterвҖ“Gummel finite elements on a tanh-clustered multilayer grid for space; `scipy.integrate.solve_ivp(Radau)` for time. Poisson is a tridiagonal solve with harmonic-mean face permittivity, evaluated once per RHS call (no inner Newton loop). State vector is `y = (n, p, P)` per grid node.

**MaterialArrays cache (hot path).** `solver/mol.py:build_material_arrays(x, stack)` returns an immutable `MaterialArrays` bundle holding every per-node / per-face array the RHS needs (`D_n_face`, `D_p_face`, `eps_r`, `N_A`, `N_D`, `P_ion0`, interface masks, boundary concentrations) plus a pre-factored Poisson operator. Every experiment builds this once per run and threads it through `assemble_rhs`, `run_transient`, `split_step`, `_compute_current`, and `_integrate_step` via the `mat=` kwarg. Rebuilding it per RHS call was the dominant cost before the cache landed вҖ” do not reintroduce a code path that calls `build_material_arrays` inside the inner loop. Degradation is the one exception: it rebuilds `mat` only when the absorber-damage state actually changes (`mat_active` in `run_degradation`).

**Poisson LU cache.** `physics/poisson.py:factor_poisson(x, eps_r)` does one LAPACK `dgttrf` call and returns a `PoissonFactor` holding `(dl, d, du, du2, ipiv)`. `solve_poisson_prefactored(fac, rho, phi_left, phi_right)` then runs a single `dgttrs` per call вҖ” ~40Г— faster than the legacy `solve_poisson` path. The factorization only depends on `(x, eps_r)`, which are constant across a transient, so it is cached inside `MaterialArrays.poisson_factor`. Legacy `solve_poisson` is kept as a fallback but is not on any hot path.

**Heterojunctions.** `chi` (electron affinity, eV) and `Eg` (band gap, eV) on each layer set the band offsets. Interface recombination uses per-interface `(v_n, v_p)` surface-recombination velocities carried by `DeviceStack.interfaces`.

**Band-offset contact BCs.** `DeviceStack.compute_V_bi()` derives the built-in potential from the Fermi-level difference across the heterostack (accounting for chi, Eg, doping, and ni). When chi=Eg=0 in all layers (legacy configs), it falls back to the manual `V_bi` field. The computed value is stored as `MaterialArrays.V_bi_eff` and used for voltage sweep range calculation (`V_max` defaults). The Poisson BC still uses `stack.V_bi` to match IonMonger's convention вҖ” do not substitute `V_bi_eff` into the Poisson boundary without careful validation, as IonMonger treats V_bi as a free parameter representing the degenerate-doping limit.

**Thermionic emission (TE) at heterointerfaces.** At interfaces where the conduction-band offset |delta_Ec| or valence-band offset |delta_Ev| exceeds 0.05 eV, the SG flux is capped to the Richardson-Dushman thermionic emission limit (`fe_operators.thermionic_emission_flux`). This prevents the SG scheme from overestimating current across sharp band discontinuities (a known artifact when the band offset is resolved in a single grid spacing). The capping is applied in `continuity.py:carrier_continuity_rhs`. Richardson constants `A_star_n` and `A_star_p` default to the free-electron value (1.2017e6 A/(mВІВ·KВІ)) and can be overridden per layer in YAML configs. Interface faces where TE activates are pre-computed in `MaterialArrays.interface_faces`. Note: IonMonger does not use TE, so our V_oc is ~0.1 V higher than IonMonger's on the same parameter set; Phase 5 (tiered modes) will add a legacy mode that disables TE for exact IonMonger reproduction.

**Transfer-matrix optics (TMM).** `physics/optics.py` implements the coherent thin-film transfer-matrix method (Pettersson et al. 1999 / Burkhard et al. 2010) for position-resolved optical generation G(x). Each layer carries `optical_material` (string key for n,k CSV data in `perovskite_sim/data/nk/`) or `n_optical` (constant refractive index fallback). When any layer has `optical_material` set, `mol.py:_compute_tmm_generation` builds the TMM stack, loads the AM1.5G spectrum from `data/am15g.csv`, and computes G(x) once during `build_material_arrays`. The result is stored as `MaterialArrays.G_optical` and used in `assemble_rhs` instead of Beer-Lambert. Key physics: the absorption formula includes the `n_real / n_ambient` Poynting-vector correction so that R+T+A=1 (energy conservation). Without `optical_material`, the original Beer-Lambert `G = alpha * Phi * exp(-alpha*x)` path is unchanged. The `_inv2x2` batched matrix inverse in `optics.py` has a determinant guard (clamps `|det| < 1e-30` to avoid division by zero at wavelengths where the transfer matrix is singular); do not remove this guard.

**Activated presets:** `nip_MAPbI3_tmm.yaml`, `pin_MAPbI3_tmm.yaml` (Phase 2 вҖ” Apr 2026). Both prepend a 1 mm `role: substrate` glass layer (optical-only) and set `optical_material` on every electrical layer. The vanilla `nip_MAPbI3.yaml` and `pin_MAPbI3.yaml` remain Beer-Lambert for back-compat with existing benchmarks. Substrate layers are filtered out of the drift-diffusion grid by `electrical_layers()` in `models/device.py`; the TMM spatial grid is offset by the substrate cumulative thickness so G(x) lands on the correct electrical nodes.

**Photon recycling / detailed-balance radiative recombination (Phase 3.1 вҖ” Apr 2026).** When `SimulationMode.use_photon_recycling` is True and TMM optics are active, `build_material_arrays` scales the absorber `B_rad` by the Yablonovitch single-pass escape probability `P_esc = min(1, 1/(4В·nВІВ·ОұВ·d))`, where `Оұ(О»_gap) = 4ПҖk(О»_gap)/О»_gap` is the intrinsic absorption at the band edge and `n` is the real index at the same wavelength (both read from the TMM layer's n,k arrays at `О»_gap = hc/Eg`). The factor is applied per absorber layer using the role tag, so only layers marked `role: absorber` get rescaled вҖ” transport layers are untouched. Implementation lives in `physics/photon_recycling.py` (`compute_p_esc`, `compute_p_esc_for_absorber`, `wavelength_at_gap`). The flag is on in both FAST and FULL, off in LEGACY. Comparing PR on vs off is only apples-to-apples when TMM is on in both runs (G(x) must be bit-identical); the regression test `tests/regression/test_photon_recycling_voc.py` does this by `monkeypatch`-ing `resolve_mode` on both `models.mode` and `solver.mol` before each V_oc measurement. The `radiative_limit.yaml` preset (absorber П„ вҶ’ 1 s, Auger = 0, detailed-balance `B_rad = 4e-17`) is the canonical test vector and produces a ~78 mV V_oc boost with PR on, inside the literature window for MAPbI3. The reciprocal emission leg вҖ” adding the reabsorbed fraction back into G(x) self-consistently вҖ” landed as Phase 3.1b (see below); when `use_radiative_reabsorption` is on, the build-time `B_rad *= P_esc` scaling is skipped and the suppression is delivered per-RHS instead, which is bit-equivalent at V_oc but more accurate when nВ·p is non-uniform.

**Self-consistent radiative reabsorption (Phase 3.1b вҖ” Apr 2026).** Closes the photon-recycling loop opened in Phase 3.1. When `SimulationMode.use_radiative_reabsorption` is True (FULL only вҖ” FAST keeps the cheaper Phase 3.1 build-time scaling), `build_material_arrays` does **not** apply the `B_rad *= P_esc` collapse; instead, for every absorber layer with `Eg > 0` it caches a `(mask, P_esc, thickness)` tuple on `MaterialArrays.absorber_masks / absorber_p_esc / absorber_thicknesses` and sets `has_radiative_reabsorption = True`. On every RHS call `assemble_rhs` then computes per absorber `R_tot = вҲ« B_rad В· n В· p dx` over the cached mask and adds the reabsorbed fraction `G_rad = R_tot В· (1 вҲ’ P_esc) / thickness` uniformly back onto the absorber's `G` nodes. Net effect: the bulk `B_rad` on the cache stays at its intrinsic value, full radiative emission acts as a sink in `carrier_continuity_rhs`, and the Yablonovitch-trapped fraction acts as a uniform source. In the spatially-uniform nВ·p limit (which is exactly what `V_oc` looks like in the bulk because the quasi-Fermi splittings are flat away from the contacts) this reproduces the Phase 3.1 V_oc boost to within sub-mV вҖ” the equivalence pinned by `tests/regression/test_radiative_reabsorption.py::test_radiative_reabsorption_matches_phase_3_1_at_voc`. Where 3.1b differs is the non-uniform regime: under strong injection, near the contacts, or in tandem sub-cells with engineered absorber profiles, emission near a high-`nВ·p` region can feed generation in a low-`nВ·p` region, redistributing carriers in a way the build-time scaling cannot capture. Like Phase 3.2 and 3.3 this hook intentionally breaks the "G computed once at build time" invariant вҖ” but only on absorber nodes where `has_radiative_reabsorption` is True, so the cost is bounded. `assemble_rhs` always copies `G` before adding the source so `mat.G_optical` stays read-only across RHS calls. The flag is gated under `use_photon_recycling`: turning radiative reabsorption on without photon recycling is a no-op (no `P_esc` was ever computed). The backend label appends `В· PR reabsorption` when active, distinguishing FULL from FAST in the SSE result payload (`backend/main.py:_describe_active_physics`).

**Field-dependent mobility (Phase 3.2 вҖ” Apr 2026).** When `SimulationMode.use_field_dependent_mobility` is True and at least one layer sets a nonzero `v_sat_{n,p}` or `pf_gamma_{n,p}`, `assemble_rhs` recomputes effective per-face diffusion coefficients on every RHS call from the Poisson-solved face field `E_face = -вҲӮПҶ/вҲӮx`. Two empirical models compose multiplicatively: Poole-Frenkel `Ој_PF(E) = ОјвӮҖ В· exp(Оі_PF В· вҲҡ|E|)` (field-assisted hopping in disordered / organic HTLs вҖ” Оі ~ 3e-4 (V/m)^-0.5 typical for spiro-OMeTAD) followed by Caughey-Thomas `Ој_CT(E) = Ој В· (1 + (ОјВ·|E|/v_sat)^ОІ)^(-1/ОІ)` (velocity-saturation cap вҖ” ОІ = 2 is the Canali form for silicon electrons, ОІ = 1 is the Thornber form for silicon holes; v_sat ~ 1e5 m/s is perovskite-realistic). Both primitives live in `physics/field_mobility.py` (`caughey_thomas`, `poole_frenkel`, `apply_field_mobility`). **Critically, this is the first physics path that intentionally breaks the "build once, reuse" invariant of `MaterialArrays`** вҖ” Ој(E) depends on state, so `D_n_face` / `D_p_face` are recomputed per RHS call rather than cached. The hook is opt-in per layer: v_sat = 0 and Оі_PF = 0 (the defaults) leave Ој untouched at that face, and `MaterialArrays.has_field_mobility = False` when no layer opts in вҖ” in which case the hot path is skipped entirely and results are bit-identical to Phase 3.1. `jv_sweep.compute_current_components` applies the same per-face correction to its terminal J_n / J_p calculation so charge conservation at the contacts is preserved. The integration test `tests/integration/test_field_mobility_integration.py` exercises the ON path (default presets leave it OFF; injecting v_sat perturbs the full J-V curve measurably without any NaN/Inf).

**Tiered simulation modes (Phase 5 вҖ” Apr 2026).** Every Phase 1вҖ“3 physics upgrade is gated on a flag in `SimulationMode` (defined in `perovskite_sim/models/mode.py`). The three named tiers вҖ” `LEGACY`, `FAST`, `FULL` вҖ” bundle the nine flags into useful combinations so that callers can pick a fidelity / speed trade-off without editing individual flags. The tier acts as a **ceiling**: turning a flag on only enables the corresponding hook if the config actually provides the parameter the hook needs (TMM needs `optical_material`, TE needs non-zero `chi` and `Eg` with a real interface offset, selective contacts need at least one `S_*`, field-mobility needs `v_sat_{n,p}` or `pf_gamma_{n,p}`, radiative reabsorption needs `use_photon_recycling` plus an absorber with `Eg > 0` under TMM, etc.). This "enable-if-configured" pattern means the tier strings are safe to set on every preset вҖ” flags are silent when inactive rather than forcing physics that the stack cannot support. Flag matrix: **LEGACY** turns everything off for IonMonger reproducibility. **FAST** turns on every *build-once* upgrade (TE, TMM, dual ions, trap profile, T-scaling, photon recycling) but keeps the three *per-RHS* upgrades off вҖ” radiative reabsorption (3.1b), field-dependent mobility (3.2), and selective contacts (3.3) all break the MaterialArrays build-once invariant, so FAST tier skips them. **FULL** turns everything on, including the three per-RHS hooks. A YAML preset can pin a tier with a top-level `device.mode: legacy|fast|full`; `DeviceStack.mode` defaults to `"full"` so any config that does not pin a mode gets the full physics set. `resolve_mode` (in `models/mode.py`) is idempotent on `SimulationMode` instances, parses strings case-insensitively, and returns `FULL` for `None`. The backend renders the resolved tier into the SSE result payload via `_describe_active_physics`, which produces strings like `"FULL  band offsets В· TE В· TMM В· dual ions В· trap profile В· T-scaling В· photon recycling В· PR reabsorption В· Ој(E) В· Robin contacts"` вҖ” the per-RHS fragments only appear when the corresponding flag is on, so FAST and LEGACY get a shorter label. The tier system is invariant-tested in `tests/unit/models/test_mode.py` (flag monotonicity: `LEGACY вҠҶ FAST вҠҶ FULL`), and end-to-end in `tests/regression/test_tier_regression.py` (LEGACY reproduces FAST/FULL on a plain preset; FAST ignores per-RHS opt-ins even when the config provides them; LEGACY disables TE even with band offsets configured; YAML `mode:` round-trips through the full loader).

**Selective / Schottky outer contacts (Phase 3.3 вҖ” Apr 2026).** The pre-3.3 outer boundary treatment was a hard Dirichlet pin: at nodes 0 and N-1 the carrier densities were clamped to their ohmic-equilibrium values (`mat.n_L`, `mat.n_R`, `mat.p_L`, `mat.p_R`) and `dn` / `dp` at those indices were zeroed so the solver never touched them. This is the correct limit for an ideal ohmic contact but cannot represent a selective (electron- or hole-blocking) or Schottky-barrier contact, which are the dominant outer-contact types in real perovskite devices. When `SimulationMode.use_selective_contacts` is True and `DeviceStack` supplies at least one of `S_n_left`, `S_p_left`, `S_n_right`, `S_p_right` (in m/s), `build_material_arrays` sets `has_selective_contacts = True` and the four `S` values propagate onto `MaterialArrays`. `assemble_rhs` then (a) keeps the boundary density for that carrier/side from the integrated state vector rather than overriding it, (b) computes a Robin-type flux `J = Вұq В· S В· (density вҲ’ density_eq)` via `physics/contacts.selective_contact_flux`, and (c) plumbs that flux through `carrier_continuity_rhs` as an extra `J_n_L` / `J_n_R` / `J_p_L` / `J_p_R` entry in `params`, which replaces the zero-flux pad at that boundary and skips the `dn[0]=0` / `dn[-1]=0` pin for that specific carrier. Sides left as `None` (the default, and what every shipped preset supplies) keep the exact pre-3.3 Dirichlet behaviour, so the feature is backward-compatible at the numerics level. `S вҶ’ вҲһ` recovers the ohmic limit; `S = 0` is a perfectly blocking (Neumann) contact; finite S allows the boundary node to evolve with a Robin time constant `~ dx_cell / S`. YAML accepts both a nested `device.contacts: { left: {S_n, S_p}, right: {S_n, S_p} }` block (preferred) and flat top-level `device.S_n_left` etc. keys. `experiments/jv_sweep._state_fields`, `solver/mol.split_step`, and the sign-aware `dn[boundary] = 0` overwrites in `assemble_rhs` all respect the Robin/Dirichlet split per carrier / per side. Schottky helpers `schottky_equilibrium_n(N_c, ПҶ_B, V_T)` and `schottky_equilibrium_p(N_v, ПҶ_B, V_T)` live alongside the Robin primitives in `physics/contacts.py` вҖ” these compute the thermionic equilibrium density at a barrier and are intended as the source of the `n_eq` argument to `selective_contact_flux` when the user wants a Schottky rather than ohmic-equilibrium reference. The `configs/selective_contacts_demo.yaml` preset exercises the full path on the nip MAPbI3 stack (small wrong-sign S, moderate matched-carrier S) and the integration test `tests/integration/test_selective_contacts_integration.py` covers the activation flag, YAML round-trip, finite-RHS under aggressive Robin fluxes including the `S=0` Neumann limit, and a J-V smoke test that fails if the Robin hook is silently bypassed.

**Temperature-dependent radiative rate and bandgap (Phase 4b вҖ” Apr 2026).** Extends the Phase 4 temperature-scaling hook (`SimulationMode.use_temperature_scaling`, gated via FAST/FULL, off in LEGACY) to cover two more parameters that matter at non-300 K: the band-to-band radiative coefficient `B_rad` and the optical bandgap `Eg`. Both are opt-in via new `MaterialParams` fields so pre-Phase-4b configs are bit-identical. `B_rad_at_T(B_300, T, gamma)` implements the empirical power law `B(T) = B_300 В· (T/300)^Оі` with `Оі = 0` as the opt-out default (literature typical for MAPbI3 is `Оі вүҲ -1.5`, the Varshni-free detailed-balance scaling for a non-degenerate semiconductor at fixed bandgap); setting `MaterialParams.B_rad_T_gamma` to a non-zero value activates the scaling in `build_material_arrays` on every per-layer `B_rad` write. `eg_at_T(Eg_300, T, alpha, beta)` implements the Varshni bandgap shift **referenced to 300 K** (rather than the classical Varshni-to-0-K form), so `MaterialParams.Eg` continues to mean "the bandgap at 300 K" and the shift is applied only when `varshni_alpha вү  0`: `Eg(T) = Eg_300 вҲ’ ОұВ·[TВІ/(T+ОІ) вҲ’ T_REFВІ/(T_REF+ОІ)]`. Silicon uses `Оұ вүҲ 4.73e-4 eV/K, ОІ вүҲ 636 K` (heating narrows Eg); MAPbI3 is opposite вҖ” Eg *widens* with heating, reproduced by `Оұ вүҲ вҲ’3e-4 eV/K, ОІ вүҲ +200 K`. The Varshni-shifted `Eg_T` is fed **both** to `chi[mask]`/`Eg[mask]` (so downstream thermionic emission, band offsets, and photon recycling see the shifted band edges) **and** to `ni_at_T` (so `niВІ(T) вҲқ exp(вҲ’Eg(T)/kT)` is self-consistent with the shifted bandgap). The degenerate case `T + ОІ = 0` returns `Eg_300` rather than dividing by zero. Both hooks respect the LEGACY gate вҖ” if `sim_mode.use_temperature_scaling` is off, `B_rad` and `Eg` stay at their 300 K values even when the new fields are populated (pinned by the `test_legacy_mode_*` tests in `tests/unit/solver/test_temperature_scaling_plumbing.py`). The full test surface is 14 unit tests in `tests/unit/physics/test_temperature.py` (identity-at-300 K, power-law exponent, zero-input sentinels, default-gamma-is-zero, Varshni sign checks for silicon vs MAPbI3 parameter sets, degenerate-denominator guard) plus 7 plumbing tests in `tests/unit/solver/test_temperature_scaling_plumbing.py` (default leaves arrays unchanged, opt-in scales correctly, legacy-mode freezes at 300 K, shipped configs remain bit-identical to the pre-Phase-4b build path). Phase 4b does not add a new tier flag вҖ” it reuses the existing `use_temperature_scaling` gate, which keeps LEGACY bit-identical to IonMonger even when a preset carelessly sets `B_rad_T_gamma` or `varshni_alpha`.

**Position-dependent trap profiles (Phase 4a вҖ” Apr 2026).** The bulk SRH lifetime on each layer represents the cleanest part of the film; in real perovskite devices grain-boundary defects concentrate near each transport-layer interface and the local SRH lifetime drops accordingly. Phase 4a turns the original inline Phase 4 profile into a reusable physics module and adds a second shape for defect layers with a well-defined finite extent. The primitives live in `physics/traps.py`: `exponential_edge_profile(x_local, thickness, N_t_interface, N_t_bulk, L_d)` returns `N_t(x) = N_t_bulk + (N_t_interface вҲ’ N_t_bulk) В· (e^{вҲ’x/L_d} + e^{вҲ’(dвҲ’x)/L_d})` (Phase 4 baseline вҖ” two exponentials added rather than maxed so a thin layer with overlapping decay tails smoothly saturates at `N_t_interface` instead of double-counting one side); `gaussian_edge_profile(...)` returns the same construction with `G(s) = exp(вҲ’(s/sigma)ВІ)` and falls off faster in the tail (`~1e-4` at 3Пғ vs `~0.05` at 3В·L_d for the exponential) вҖ” useful when the interface defect slab has a measured finite thickness rather than a long tail; `tau_from_trap_density(tau_bulk, N_t, N_t_bulk)` applies the SRH inverse-density rule `tau(x) = tau_bulk В· N_t_bulk / N_t(x)` with the denominator floored at `N_t_bulk` so that `N_t_interface < N_t_bulk` (a passivation profile) still yields a physical, monotone correction; `has_trap_profile_params(params)` is the presence check used by `build_material_arrays` to decide whether to call the profile. Shape selection is per-layer via the new `MaterialParams.trap_profile_shape` field (`"exponential"` default | `"gaussian"`), wired through `config_loader.py` as `layer_cfg.get("trap_profile_shape", "exponential")`. `L_d = 0` or `sigma = 0` are degenerate "no profile" sentinels that return `N_t_bulk` everywhere, and `L_d = 0` on a layer that still opts in simply leaves `tau` at its layer-uniform bulk value. `build_material_arrays` also caches the per-node trap density on `MaterialArrays.N_t_node` (zeros on nodes that belong to a layer with no trap params, `N_t_bulk` on layers that only specify the bulk density without a profile, and the full `N_t(x)` on layers that opt in), plus a boolean `has_trap_profile` that reports whether at least one layer triggered the profile. These diagnostics let downstream hooks (interface SRH, band-tail absorption, mobility degradation) reuse the same `N_t(x)` without recomputing the profile. The whole module is gated by `SimulationMode.use_trap_profile` (on in FAST and FULL, off in LEGACY); when the flag is off `build_material_arrays` skips the profile entirely and `N_t_node` reflects the uniform-bulk fallback. Unit tests in `tests/unit/physics/test_traps.py` pin the endpoint formula, midpoint bulk recovery, monotonic decay into the interior, Gaussian tail decays faster than exponential at ~3В·L, zero-length sentinel yields flat bulk, `tau_from_trap_density` passivation floor, and the `has_trap_profile_params` presence check. The existing integration test `tests/integration/test_temperature_trap.py` continues to cover the end-to-end `build_material_arrays` path (now through the refactored module) without modification.

**Experiments** (`perovskite_sim/experiments/`):
- `jv_sweep.run_jv_sweep` вҖ” forward then reverse scan; reuses the previous steady state as initial condition so ionic memory is preserved and the hysteresis loop comes out of the physics, not post-processing. A per-step `_JV_RADAU_MAX_NFEV = 100_000` guard aborts any single voltage step that consumes too many RHS evaluations (prevents the solver from hanging on pathological substrate-stack configs). Supports `illuminated=False` for dark J-V (G=0, dark-equilibrium start) and `save_snapshots=True` for collecting `SpatialSnapshot` at every voltage point.
- `jv_sweep.compute_current_components` вҖ” decomposes the terminal current into J_n (electron), J_p (hole), J_ion (ionic, both species), and J_disp (displacement) at every mesh face. Uses the same SG flux formulas as `assemble_rhs` but multiplied by Q for A/mВІ units. Ion current includes the steric Blakemore correction and dual-ion support.
- `jv_sweep.extract_spatial_snapshot` вҖ” extracts a `SpatialSnapshot` (phi, E, n, p, P, rho) from a packed state vector at a given voltage. Used internally by `save_snapshots` and available as a standalone API.
- `impedance.run_impedance` вҖ” at each frequency integrates a few AC cycles and extracts amplitude/phase with a lock-in (sin/cos multiply + low-pass). Adds the displacement current `ОөвӮҖВ·Оө_rВ·вҲӮE/вҲӮt`.
- `degradation.run_degradation` вҖ” long-time transient; at each probe time it takes a **frozen-ion snapshot**: a `replace`-d copy of the stack with `D_ion = 0` in every layer is used for a short settle integration at each probe voltage (`_freeze_ions` + `_measure_snapshot_metrics`). This measures the instantaneous electronic response under the current ionic configuration and is the only correct way to compute snapshot JвҖ“V without averaging over ion drift.
- `tpv.run_tpv` вҖ” transient photovoltage experiment. Equilibrates at V_oc under illumination, applies a fractional generation pulse (delta_G_frac), tracks J(t) at fixed V_app=V_oc, converts to V(t) via the small-signal relation delta_V = -J * R_oc, and fits a mono-exponential decay time tau. Returns a `TPVResult` with t, V, J arrays plus V_oc, tau, delta_V0.
- `tandem_jv.run_tandem_jv` вҖ” 2T monolithic tandem driver. Runs a combined TMM over the full stack (`physics/tandem_optics.py`) to partition G(x) into top/bottom sub-cell profiles, then performs independent drift-diffusion JвҖ“V sweeps with `fixed_generation`, and series-matches at a common current grid (`series_match_jv`). Returns a `TandemJVResult` holding per-sub-cell results plus the tandem JвҖ“V curve.

**Tandem optics** (`physics/tandem_optics.py`): Runs one TMM over the entire tandem stack (top + junction + bottom), then splits absorption by layer. Junction layers count as parasitic absorption. The `TandemGeneration` dataclass holds `G_top(x)` and `G_bot(x)` arrays. The tandem config model lives in `models/tandem_config.py` (`TandemConfig`, `JunctionLayer`).

**2D extension вҖ” Stage A lateral-uniform parity (Phase 6 вҖ” Apr 2026).** Lives in `perovskite_sim/twod/` as a parallel sub-package вҖ” it does not touch the 1D solver hot path. The 2D state is `(n(y, x), p(y, x))` only; ions are held as a static Poisson background (`P_ion_static_1d`) extruded from the 1D illuminated steady state, so Stage A has no ionic memory across voltage steps. Grid: `build_grid_2d` produces a tensor-product grid with the existing tanh-clustered y-axis (one `Layer(thickness, Ny_per_layer)` per electrical layer) and a uniform x-axis of `Nx + 1` nodes spanning `lateral_length`. Poisson: `physics/poisson_2d.py` assembles a sparse 5-point stencil with harmonic-mean face permittivity and `scipy.sparse.linalg.splu` factor cached on `MaterialArrays2D` вҖ” the same Dirichlet contacts as 1D (`phi[0,:]=0`, `phi[-1,:]=V_bi - V_app`) and lateral periodic or Neumann BCs. The Poisson RHS includes the frozen-ion term `Q*(P_ion_static - P_ion0_2d)`, which is critical for matching 1D's $\rho$ вҖ” without it the 2D $\varphi$ diverges at heterointerfaces and SG fluxes blow up. Continuity: `twod/flux_2d.py` implements vectorised 2D Scharfetter-Gummel for both directions with **harmonic-mean face-D** averaging (matches 1D `MaterialArrays.D_n_face`); the SG flux uses `phi_n = phi + chi`, `phi_p = phi + chi + Eg` so band offsets are preserved at heterointerfaces. Thermionic-emission capping: `twod/continuity_2d.py` caps `Jy_n` / `Jy_p` at vertical heterointerfaces (where `|delta_Ec| > 0.05` eV or `|delta_Ev| > 0.05` eV) using the same Richardson-Dushman bound as 1D `continuity.py`. The `dn = +(div_x_n + div_y_n)/Q + G - R` sign for electrons (charge $-q$) and `dp = -(div_x_p + div_y_p)/Q + G - R` for holes are pinned by `tests/regression/test_twod_validation.py` вҖ” getting either sign wrong produces immediate Radau failure at V=0.

The Stage-A J-V driver `twod/experiments/jv_sweep_2d.run_jv_sweep_2d` bootstraps via 1D `solve_illuminated_ss` (or `solve_equilibrium` for dark) on the same y-grid, freezes the equilibrated ion profile via `P_ion_static_1d`, broadcasts $(n, p)$ to 2D, and walks the voltage grid with a per-step `run_transient_2d` settle (Radau-LU on the ~250Г—250 system at the test-grid sizes). The validation gate `tests/regression/test_twod_validation.py` runs the 1D and 2D solvers on the Beer-Lambert MAPbI3 preset with frozen ions and matched grids (1D `N_grid=31`, 2D `Ny_per_layer=10` Г— 3 electrical layers + 1 = 31) and asserts: $|V_{oc}^{2D} - V_{oc}^{1D}| \le 10^{-4}$ V, $|\Delta J_{sc}|/|J_{sc}| \le 5\times10^{-4}$, $|\Delta \text{FF}| \le 10^{-3}$, lateral invariance of $n$ at $V_{oc}$ to $\le 10^{-9}$, divergence of total electron+hole current bounded at interior nodes, and Poisson Dirichlet BCs match exactly. Final passing values: bit-identical 1D and 2D (`V_oc=910.634` mV, `J_sc=424.036` A/mВІ, FF=0.8247).

Backend dispatch: `kind="jv_2d"` in `backend/main.py` accepts `lateral_length`, `Nx`, `Ny_per_layer`, `V_max`, `V_step`, `settle_t`, `illuminated`, `lateral_bc`, `save_snapshots` and serialises `JV2DResult` (V, J, grid_x in nm, grid_y in nm, lateral_bc, snapshots). The frontend pane `frontend/src/workstation/panes/jv-2d-pane.ts` is registered under the `'2D / Microstructural (Stage A)'` group as `JвҖ“V Sweep (2D)`; the result renderer in `main-plot-pane.ts` flips the J sign to match the 1D solar convention. The TMM preset `nip_MAPbI3_tmm` and the BL preset `nip_MAPbI3` (and the alias `configs/twod/nip_MAPbI3_uniform.yaml`) are the supported Stage-A presets вҖ” the Stage-A regression uses BL because the TMM preset has the pre-existing 1D coupled-solver regression at V=0.21 V (see `project_tmm_jv_regression_021.md`).

Stage-A scope boundaries вҖ” Phase 6 deliberately does **not** include 2D ions, 2D radiative reabsorption / field mobility / Robin contacts, or per-grain $\tau$ profiles. Those reuse the same `MaterialArrays2D` cache slots (which already hold `chi`, `Eg`, `A_star_n/p`, `interface_y_faces`, etc.) and the same `assemble_rhs_2d` skeleton, but are out of scope for the Stage-A validation gate.

**2D Microstructure вҖ” Stage B (Phase 6 вҖ” Apr 2026).** Lateral grain boundaries with reduced SRH lifetimes, painted onto the absorber via `perovskite_sim/twod/microstructure.py`. The data model is two frozen dataclasses: `GrainBoundary(x_position, width, tau_n, tau_p, layer_role)` represents one vertical band, and `Microstructure(grain_boundaries=...)` is the container. `build_tau_field(grid, ustruct, tau_n_bulk_per_y, tau_p_bulk_per_y, layer_role_per_y)` extrudes the bulk $\tau$ along $x$ and then overrides cells inside each GB band where `|x - gb.x_position| < gb.width/2` AND the role tag at that $y$ matches `gb.layer_role`. Empty `Microstructure()` reduces to identity extrusion (Stage-A bit-identical). The YAML schema is a top-level `microstructure:` block with strict-key validation (`load_microstructure_from_yaml_block` raises `ValueError` on unknown keys so typos surface immediately):

```yaml
microstructure:
  grain_boundaries:
    - x_position: 250e-9   # m, GB centre
      width:      5e-9     # m, GB band width
      tau_n:      5e-8     # s, electron SRH lifetime inside GB
      tau_p:      5e-8     # s, hole SRH lifetime inside GB
      layer_role: absorber # only nodes tagged this role get overridden
```

`load_device_from_yaml` parses the block and stores the result on `DeviceStack.microstructure`. `run_jv_sweep_2d` accepts `microstructure: Microstructure | None = None`; when None it picks up `stack.microstructure` (then falls back to `Microstructure()`). Inside `build_material_arrays_2d` the role mask is derived by `_layer_role_at_each_y`, which assigns each $y$ node to exactly one electrical layer; layer-boundary rows are tagged with the upstream layer (HTL on the HTL/absorber interface, absorber on the absorber/ETL interface), so a single absorber GB touches the absorber/ETL interface row but not the HTL/absorber row. The shipped Stage-B preset `configs/twod/nip_MAPbI3_singleGB.yaml` (one centred GB, $\tau_\text{GB}=50$ ns vs $\tau_\text{bulk}=1$ Вөs, width 5 nm) drops $V_\text{oc}$ by ~46 mV vs the lateral-uniform baseline вҖ” pinned to a 5вҖ“100 mV regression window in `tests/regression/test_twod_microstructure.py`. Headline experiment: `voc_grain_sweep` (in `twod/experiments/voc_grain_sweep.py`) sweeps a sequence of grain sizes $L_g$, with one centred absorber GB per $L_g$ and periodic lateral BCs, and returns $V_\text{oc}(L_g)$, $J_\text{sc}(L_g)$, $\text{FF}(L_g)$ вҖ” exposed as `kind="voc_grain_sweep"` from the backend and as the **V_oc(L_g) Grain Sweep** entry in the workstation. The `jv_2d` backend kind also accepts an optional `params.microstructure` block (same YAML shape) so users can add a single GB to a Stage-A run from the UI without editing YAML.

Stage-B scope boundaries вҖ” only $\tau$ heterogeneity is wired through; per-grain mobility, anisotropy, and dual-ion segregation across grains are deferred. The Stage-B physics gate uses Beer-Lambert presets to avoid the pre-existing 1D coupled-solver regression at V=0.21 V on TMM presets (`project_tmm_jv_regression_021.md`).

**2D Robin/selective contacts вҖ” Stage B(c.1) (Phase 6 вҖ” Apr 2026).** Ports the 1D Phase 3.3 Robin/selective-contact boundary condition to `assemble_rhs_2d`. When `SimulationMode.use_selective_contacts` is True and `DeviceStack` supplies any of `S_n_left`, `S_p_left`, `S_n_right`, `S_p_right`, `build_material_arrays_2d` sets `MaterialArrays2D.has_selective_contacts = True` and maps the four scalar S values via the 1D-transport-axis bridge:

```
DeviceStack.S_n_left  вҶ’ MaterialArrays2D.S_n_top  (y=0,    HTL / p-contact)
DeviceStack.S_p_left  вҶ’ MaterialArrays2D.S_p_top
DeviceStack.S_n_right вҶ’ MaterialArrays2D.S_n_bot  (y=Ny-1, ETL / n-contact)
DeviceStack.S_p_right вҶ’ MaterialArrays2D.S_p_bot
```

`continuity_rhs_2d` computes a one-sided Neumann y-divergence at the boundary rows (previously skipped). `assemble_rhs_2d` guards on `has_selective_contacts`: the `else` branch still zeros the four Dirichlet rows (Stage A/B(a) behaviour unchanged); the `if` branch calls `_apply_robin_contacts_2d` which subtracts the Robin wall flux from the Neumann base. The sign table (verified from first principles; `dp` signs are opposite to `dn` because `dp = вҲ’div_p/Q`):

```
dn[0,  :] -= J_n_top / (Q * hy_top)   # top electrons: J_n_top = +qВ·SВ·(nвҲ’n_eq)
dp[0,  :] += J_p_top / (Q * hy_top)   # top holes:     J_p_top = вҲ’qВ·SВ·(pвҲ’p_eq)  вҶҗ +=
dn[-1, :] += J_n_bot / (Q * hy_bot)   # bot electrons: J_n_bot = вҲ’qВ·SВ·(nвҲ’n_eq)
dp[-1, :] -= J_p_bot / (Q * hy_bot)   # bot holes:     J_p_bot = +qВ·SВ·(pвҲ’p_eq)  вҶҗ -=
```

Equilibrium densities use `mat.n_eq_left` / `mat.p_eq_left` for y=0 and `mat.n_eq_right` / `mat.p_eq_right` for y=Ny-1 (existing `(Nx,)` arrays; note the code comments on these fields had "top/bottom" reversed вҖ” fixed in this stage). The parity gate `tests/regression/test_twod_validation.py::test_twod_robin_parity_vs_1d` runs 1D Phase 3.3 vs 2D Stage B(c.1) on `configs/selective_contacts_demo.yaml` (frozen ions) and pins |О”V_oc| вүӨ 1 mV вҖ” final passing values are bit-identical at the displayed precision (V_oc=910.634 mV, J_sc=424.036 A/mВІ, FF=0.8247). The bounded-shift sanity test uses an aggressively-blocking Robin stack (`S_n_left=1e-4, S_p_left=1e-3, S_n_right=1e-3, S_p_right=1e-4` m/s вҖ” same envelope as 1D `test_selective_contacts_integration`) and asserts |О”V_oc| вҲҲ [1, 150] mV vs the Dirichlet baseline; the demo preset's matched S=1e3 m/s is effectively ohmic at this device thickness so the demo would not be useful for this assertion. `selective_contact_flux` was extended to accept array-valued `density_eq` so per-x equilibrium variation is supported when the build path eventually paints lateral structure into `n_eq_left/right`. Stage B(c.1) does not add Schottky barriers, per-column S patterning, or changes to the backend/frontend вҖ” those are out of scope.

**2D field-dependent mobility вҖ” Stage B(c.2) (Phase 6 вҖ” Apr 2026).** Ports the 1D Phase 3.2 Ој(E) hook to `assemble_rhs_2d` using a **face-normal** formulation: x-faces use only `|E_x_face|`, y-faces use only `|E_y_face|`. When `SimulationMode.use_field_dependent_mobility` is True AND any layer sets a non-zero `v_sat_{n,p}` or `pf_gamma_{n,p}`, `build_material_arrays_2d` sets `MaterialArrays2D.has_field_mobility = True` and pre-computes 18 face arrays of `v_sat / ct_beta / pf_gamma` via **arithmetic mean** (interior x-faces, interior y-faces, periodic-x wrap face). On every RHS call, `assemble_rhs_2d` then calls `recompute_d_eff_2d` (in `perovskite_sim/twod/field_mobility_2d.py`) which: (1) computes `E_x_face = -(phi[:, 1:] - phi[:, :-1]) / dx[None, :]` and analogously `E_y_face` and the periodic wrap `E_x_wrap`; (2) forms base mobility `Ој_base = harmonic_mean(D_node) / V_T` per face (Einstein); (3) applies the existing 1D `apply_field_mobility(Ој_base, |E|, v_sat, beta, gamma_pf)` primitive face-normally вҖ” x-faces with `|E_x_face|`, y-faces with `|E_y_face|`. The primitive uses `np.abs(E)` internally so the input sign is irrelevant. (4) Recovers `D_eff = Ој_eff В· V_T` via Einstein. The effective per-face D is then forwarded through `continuity_rhs_2d` to `sg_fluxes_2d_n/p` via new keyword-only override kwargs (`D_n_x_face`, `D_n_y_face`, `D_p_x_face`, `D_p_y_face`, plus `D_n_wrap`/`D_p_wrap` for periodic). When the override is `None` (default) the harmonic-mean path runs unchanged вҶ’ bit-identical to Stage A / B(a) / B(c.1). The `D_n` / `D_p` cache stays at the constant-mobility limit; only the per-face D injected into the flux helpers carries the field correction.

**Mean choice.** `D_n` / `D_p` continue to use harmonic mean (matches `sg_fluxes_2d_*` and 1D); `v_sat` / `ct_beta` / `pf_gamma` use arithmetic mean (avoids harmonic suppression at one-side-zero, which would silently disable CT/PF at heterointerfaces).

**Activation gate.** `_has_field_mobility = sim_mode.use_field_dependent_mobility AND any(v_sat>0 or pf_gamma>0)` вҖ” same tier-as-ceiling pattern established by Stage B(c.1). LEGACY tier disables the hook even with params set; FAST tier currently lists Ој(E) among the per-RHS hooks it skips, so FAST also stays on the constant-D path.

**Validation.** Lateral-uniform 2D with `v_sat=1e2` matches 1D Phase 3.2 to within sub-mV V_oc / 5e-4 J_sc / 1e-3 FF вҖ” measured deltas are bit-identical at the displayed precision (V_oc=910.6340 mV, J_sc=423.8237 A/mВІ, FF=0.824734 in both 1D and 2D; О”V_oc вүҲ вҲ’0.0000 mV, О”J_sc/J_sc вүҲ +8e-10, О”FF вүҲ вҲ’4e-10). Pinned by `tests/regression/test_twod_validation.py::test_twod_field_mobility_parity_vs_1d`. A bounded-shift sanity test (`test_twod_field_mobility_bounded_shift`) asserts `max(|О”J|/|J|) > 1e-6` between Ој(E) on and off вҖ” measured shift is ~7Г—10вҒ»вҒҙ on this Beer-Lambert MAPbI3 stack, modest because the absorber dominates the field-mobility correction while the flat-band regions near contacts contribute little. **Out of scope.** Total-|E| (Option B) with cross-axis `E_x` / `E_y` interpolation is explicitly deferred вҖ” see `docs/superpowers/specs/2026-04-29-2d-stage-b-c2-field-mobility-design.md`. **2D self-consistent radiative reabsorption вҖ” Stage B(c.3) (Phase 6 вҖ” Apr 2026).** Ports the 1D Phase 3.1b reabsorption hook to `assemble_rhs_2d`. When `SimulationMode.use_radiative_reabsorption` is True AND `SimulationMode.use_photon_recycling` is True AND `MaterialArrays.has_radiative_reabsorption` is True (the 1D side has populated per-absorber tuples under TMM optics), `build_material_arrays_2d` translates `mat1d.absorber_masks` to 2D `absorber_y_ranges_2d` (each is a contiguous `(y_lo, y_hi)` half-open range derived from `layer_role_per_y == "absorber"`) and pre-computes `absorber_areas_2d = thickness Г— lateral_length` per absorber.

On every RHS call, `assemble_rhs_2d` calls `recompute_g_with_rad_2d` (in `perovskite_sim/twod/radiative_reabsorption_2d.py`) which: (1) for each absorber, computes `emission = B_rad[y_lo:y_hi, :] В· n[y_lo:y_hi, :] В· p[y_lo:y_hi, :]` of shape `(n_y_abs, Nx)`; (2) trapezoid in y first (axis=0) вҶ’ `(Nx,)`; then trapezoid in x вҶ’ scalar `R_tot_2D`; (3) skips if `R_tot_2D вүӨ 0`, `area вүӨ 0`, `P_esc вүҘ 1`, or fewer than 2 nodes on either axis; (4) computes `G_rad = R_tot_2D В· (1 вҲ’ P_esc) / area`; (5) augments `G[y_lo:y_hi, :] += G_rad` (uniform redistribution over absorber 2D area). The cached `mat.G_optical` is **never mutated** вҖ” the helper returns a NEW `(Ny, Nx)` array. The augmented `G_to_use` is then forwarded to `continuity_rhs_2d`. When the flag is False, `G_to_use is mat.G_optical` (same Python object) вҖ” bit-identical to Stage A / B(a) / B(c.1) / B(c.2).

**Bit-equivalent to 1D in the lateral-uniform limit.** With constant `dx`, the lateral trapezoid evaluates to `lateral_length Г— emission_y(y)`, and dividing by `area = thickness Г— lateral_length` recovers `emission_y / thickness` вҖ” exactly 1D's `G_rad = R_tot В· (1 вҲ’ P_esc) / thickness`. The 2D y-range derives from `layer_role_per_y == "absorber"` (one half-cell narrower than the 1D inclusive mask at the HTL/absorber interface, which uses upstream-tagging in 2D); this trades ~1.3 mV V_oc against the 5 mV B(c.3) parity envelope and keeps the absorber row semantics consistent with Stage B(a) microstructure painting.

**Lagged fallback in `jv_sweep_2d.run_jv_sweep_2d`.** Default per-RHS is fully self-consistent (Newton iters re-evaluate `R_tot_2D` from the live state). On `RuntimeError` from `run_transient_2d` AND `mat.has_radiative_reabsorption_2d` AND `illuminated`, `_bake_radiative_reabsorption_step_2d` evaluates `R_tot_2D` once at the entry state of the failed voltage step, folds `G_rad` into a step-local `G_optical` copy, clears the flag and zeros all four absorber tuples via `dataclasses.replace`, and retries `run_transient_2d` once with the baked `mat`. Across voltage steps the warm-start chain refreshes `R_tot` from the freshly-settled state, so the lag is bounded by `nВ·p` drift inside one settle interval. Mirrors 1D `experiments/jv_sweep.py:_bake_radiative_reabsorption_step` exactly.

**Activation gate.** `_has_rr_2d = sim_mode.use_radiative_reabsorption AND sim_mode.use_photon_recycling AND mat1d.has_radiative_reabsorption` вҖ” same tier-as-ceiling pattern established by Stage B(c.1) and B(c.2). LEGACY tier disables the hook even on TMM presets; FAST tier currently lists radiative reabsorption among the per-RHS hooks it skips, so FAST also stays on the disabled path. FULL enables when the preset supports it.

**Validation.** All three TMM-preset regression tests use a **V_oc-only warm-start** path because the underlying Stage A 2D solver cannot Newton-contract through the VвүҲ0.2 V diode-injection knee on TMM presets вҖ” see project memory `project_stage_a_2d_tmm_newton.md`. The diode I-V is approximately linear over a 10 mV window near V_oc, so the interpolation gives sub-mV V_oc agreement. J_sc and FF parity assertions are deferred until the Stage A TMM Newton issue is fixed (out of scope for B(c.3)). The shared helper is `_voc_2d_via_warm_start` in `tests/regression/test_twod_validation.py`. Tier-gate is unit-pinned by `test_legacy_mode_disables_radiative_reabsorption_in_2d` and `test_fast_mode_disables_radiative_reabsorption_in_2d`. **Out of scope.** Optical-profile-weighted redistribution (where `G_rad` would be weighted by `Оұ(x)В·I(x)` rather than uniform) is explicitly deferred. Per-grain absorber heterogeneity in the reabsorption integral, Ој(T)-coupled `B_rad(T)` beyond the existing temperature-scaling hook, and any backend / frontend changes are out of scope for Stage B(c.3).

### Solver gotcha вҖ” RHS finite-check

`assemble_rhs` includes a `np.all(np.isfinite(dydt))` guard that raises `ValueError` if any element is NaN or Inf. This catches blow-ups from singular TMM matrices, zero-thickness layers, or extreme doping imbalances early вҖ” before Radau can silently accept them and produce garbage. Do not remove this check.

### Solver gotcha вҖ” Radau max_step cap

Near flat-band (V вүҲ V_bi) the Jacobian is nearly singular and Radau's adaptive LTE estimator under-reports the error, letting it accept one giant step on the wrong branch of the implicit system. This manifests as an unphysical JвҖ“V spike (e.g. `J=258` sandwiched between `188` and `101`). **All three experiments therefore cap `max_step = О”t / k`** on every `run_transient` sub-interval (`jv_sweep._integrate_step`, `impedance`, `degradation._measure_snapshot_metrics`). Do not remove these caps; if you need tighter control, lower the divisor instead. Any new experiment that calls `run_transient` across a voltage where the Jacobian can become near-singular must apply the same cap.

## Backend (`backend/main.py`)

Thin FastAPI wrapper. Two endpoint families:

**Legacy blocking endpoints** вҖ” `POST /api/jv`, `/api/impedance`, `/api/degradation`. Run synchronously and return the full result in one response. Kept for backwards compatibility; the frontend no longer uses them.

**Streaming job endpoints** (frontend uses these):
- `POST /api/jobs` вҖ” body `{kind, config_path|device, params}`. Dispatches the experiment onto a worker thread via `JobRegistry.submit` and returns `{job_id}`. The closure captured in `_run(reporter)` passes `reporter.report` as the `progress=` kwarg to the experiment.
- `GET /api/jobs/{id}/events` вҖ” Server-Sent-Events stream. Named events are `progress` (JSON `{stage, current, total, eta_s, message}`), `result` (final serialized result), `error` (on exception), and `done` (always last). The handler uses `run_in_executor` + a 0.5 s drain timeout to avoid blocking the event loop, and emits `: keepalive` SSE comments in the gap between progress frames.

`backend/progress.py` and `backend/jobs.py` are the pub/sub primitive and the thread-per-job registry behind this. Any new experiment that wants progress should take a `progress: Callable[[str, int, int, str], None] | None = None` kwarg (see `experiments/jv_sweep.py` for the pattern) and the backend `_run` closure wraps it with `reporter.report`.

`GET /api/configs` auto-scans `configs/`, so dropping a new YAML in that folder makes it visible to the frontend with no code change.

### Development loop вҖ” restart uvicorn after backend edits

`uvicorn backend.main:app --reload` picks up most edits, but the watcher is set on the SolarLab root and occasionally misses changes under `perovskite-sim/backend/`. If `/api/jobs` or a new endpoint returns 404 after an edit, kill and restart the process rather than trusting `--reload`.

### YAML scientific-notation gotcha

PyYAML's default loader is YAML 1.1, which only treats `1.0e-9` as a float вҖ” a bare `1e-9` (no decimal point) is returned as a **string**. The frontend's numeric editor then throws `v.toExponential is not a function` when rendering such a field. The fix lives in `backend/main.py:_coerce_numbers`, a recursive pass applied in `get_config` that turns any numeric-looking string into a float. When editing the backend do not bypass this coercion, and when adding new YAML configs you can use either form, but `1e-9` is fine because the backend normalises it.

## Frontend (`frontend/src/`)

Plain TypeScript (no framework). `main.ts` wires five tabs вҖ” `JвҖ“V Sweep`, `Impedance`, `Degradation`, `Tutorial`, `Algorithm` вҖ” each mounted lazily into its panel `<section>`. Key modules:

- `api.ts` вҖ” typed fetch wrappers for the backend
- `config-editor.ts` вҖ” collapsible `<details>` layer editor with Geometry / Transport / Recombination / Ions & Optics groups. `fmt()`/`numAttr()` accept `unknown` and coerce via `Number(v)` so a non-numeric field can never crash the panel.
- `device-panel.ts` вҖ” preset dropdown + Reset button; fetches `/api/configs` and `/api/configs/{name}`
- `panels/{jv,impedance,degradation}.ts` вҖ” experiment panels: form + run button + Plotly plots + metric cards
- `panels/{tutorial,algorithm}.ts` вҖ” static documentation panels (user-guide + PDE / discretisation / solver description)
- `plot-theme.ts`, `plotly.d.ts`, `types.ts`, `ui-helpers.ts` вҖ” shared styling, type shims, small helpers

Plotly is pulled from `plotly.js-dist-min` which is why `vite build` warns about the 4 MB bundle; that warning is expected and not a bug.

**Custom stacks (Phase 2b вҖ” Apr 2026):** In full tier the Device pane renders a vertical layer visualizer with add/remove/reorder, a template library, structural validation, and a Save-As path that lands user presets in `configs/user/`. The accordion editor is preserved for fast/legacy tiers. New backend endpoints: `GET /api/layer-templates`, `POST /api/configs/user`. `GET /api/configs` now returns `{name, namespace}` entries.

### Backend URL: absolute, not proxied

`api.ts` and `job-stream.ts` both hardcode `http://127.0.0.1:8000` as the base URL. The Vite dev-server proxy is configured for `/api` in `vite.config.ts`, but its SPA history fallback intercepts `/api/configs` before the proxy has a chance to match, returning `index.html` вҖ” the JSON parse then fails with `Unexpected token '<'`. Hitting the backend directly sidesteps this. CORS is already wide open in `backend/main.py`, so this works from both `npm run dev` and `npm run preview`/`dist`.

SSE notes worth knowing:
- `EventSource` fires a native `error` event when the server closes the connection. `job-stream.ts:streamJobEvents` ignores events whose `e.data` is falsy вҖ” only SSE `event: error` frames sent from the backend are surfaced to the user.
- Progress frames and the terminal `done` frame go through named SSE events (`event: progress`, `event: result`, `event: error`, `event: done`), not the default message channel, so each has its own `addEventListener`.

### Streaming-job + progress-bar pattern

Each experiment panel (`panels/{jv,impedance,degradation}.ts`) follows the same shape:
1. Instantiate a `ProgressBarHandle` once in `mount*Panel` via `createProgressBar(progressContainer)`.
2. On button click, call `startJob(kind, device, params)` вҶ’ get `job_id`.
3. Open the stream with `streamJobEvents<TResult>(jobId, { onProgress, onResult, onError, onDone })`.
4. `onResult` calls the panel-local `render*Results`; `onDone` re-enables the Run button.

The button is disabled between `startJob` and `onDone`, and the progress bar is `reset()` on every run so a second click doesn't briefly show the previous run's "done" state.

## Configs (`configs/`)

Presets shipped with the repo:

- **Beer-Lambert:** `nip_MAPbI3`, `pin_MAPbI3`, `ionmonger_benchmark` (Courtier 2019), `driftfusion_benchmark`, `cigs_baseline` (ZnO/CdS/CIGS, `D_ion=0`), `cSi_homojunction` (n+/p wafer, `D_ion=0`)
- **TMM-enabled:** `nip_MAPbI3_tmm`, `pin_MAPbI3_tmm`, `ionmonger_benchmark_tmm`, `driftfusion_benchmark_tmm`
- **Tandem sub-cells:** `nip_wideGap_FACs_1p77` (1.77 eV top), `nip_SnPb_1p22` (1.22 eV bottom)
- **Tandem config:** `tandem_lin2019` (2T monolithic, uses `TandemConfig` not `DeviceStack`)

The YAML schema mirrors `MaterialParams` + `DeviceStack.interfaces`; see any existing file for the field list. Non-perovskite stacks must set `D_ion = 0` in every layer вҖ” the ion equations still integrate but contribute zero flux. Tandem configs use a separate `TandemConfig` schema (`models/tandem_config.py`) that references two sub-cell configs plus junction layers.

Practical solver envelope: the Radau transient handles sub-micron perovskite stacks comfortably (ionmonger reference вүҲ 25 s for a full JвҖ“V). Thick inorganic absorbers (2 Вөm CIGS, 180 Вөm Si) are structurally valid and `solve_equilibrium` converges, but a full transient JвҖ“V sweep is impractical at default tolerances. Use thinner absorbers or a coarser grid when iterating, and prefer equilibrium-level tests for those configs.

## Data Model Invariants

- `MaterialParams`, `SolverConfig`, `LayerSpec`, `DeviceStack` are **frozen dataclasses**. Never mutate in place вҖ” use `dataclasses.replace(...)` to produce a new instance (see `degradation._freeze_ions` for the canonical pattern).
- All physical quantities are SI (m, s, V, A/mВІ, mвҒ»Ві). The only exceptions are `chi` and `Eg` which are in eV (numerically equal to volts through `q`).
- `DeviceStack.interfaces` is a tuple of `(v_n, v_p)` pairs with length `len(layers) - 1`, ordered from contact side to contact side.

## Test Structure

```
tests/
в”ңв”Җв”Җ unit/          per-module physics + solver + experiments tests
в”ңв”Җв”Җ integration/   end-to-end experiment runs on preset configs
в””в”Җв”Җ regression/    physical sanity checks (V_oc range, J_sc bounds, ...)
```

`pytest` defaults (from `pyproject.toml`) exclude `-m slow`. Coverage is **opt-in** вҖ” pass `--cov=perovskite_sim --cov-report=term-missing` explicitly when you want a report. The regression suite is where to add new "result should look physically reasonable" checks.

Regression tests:
- `test_jv_regression.py` вҖ” V_oc / J_sc / FF / HI envelopes for all presets
- `test_tmm_baseline.py` вҖ” TMM-specific baselines (slow, BLAS-pinned)
- `test_conservation.py` вҖ” energy conservation (R+T+A=1) and J-V monotonicity checks; catches TMM numerical issues and solver blow-ups

### Test gotcha вҖ” slow suite BLAS thread pinning

The TMM regression suite (`tests/regression/test_tmm_baseline.py`) drives ~4700 calls to `scipy.linalg.lu_factor` on the ~300Г—300 Radau Jacobian. These matrices are too small for multi-threaded BLAS to pay off вҖ” on a 10-core box OpenBLAS spins up every LU call across all cores and the thread-creation + contention overhead turns a 14 s test into a 5-10 minute test. A Phase 2a investigation wasted four run-kills diagnosing this as a "stall" (runs were being terminated at ~4 min wall, but they would have taken another 2-3 min each to finish under oversubscription). It is NOT a hang.

`tests/conftest.py` pins BLAS threads to 1 via `threadpoolctl` whenever the `slow` marker is selected (excluding the default `-m 'not slow'` unit run). numpy/scipy must be imported inside the hook before calling `threadpool_limits`, because threadpoolctl only sees already-loaded backends. Do not remove this hook; also do not re-bake `--cov=perovskite_sim` into `pyproject.toml` `addopts` вҖ” pytest-cov's line tracer adds a further ~17Г— overhead on the same hot loop and is equally sticky (the tracer is installed during pytest-cov's own `pytest_configure`, before any user conftest gets a chance to stop it).

Canonical invocations:
- Unit + integration: `pytest` (default, ~15 s, no coverage)
- Unit with coverage: `pytest --cov=perovskite_sim --cov-report=term-missing`
- Slow regression: `pytest -m slow` (BLAS pinned automatically, ~27 s for the TMM baselines)
