# Phase E1.6 SG-face-density extraction — RFC / design spec

**Status:** DRAFT (post Phase A investigation, 2026-05-26)
**Author:** Phase E1.6 investigation spike
**Decision required:** select Phase B architecture before committing 1-2 week refactor

## Goal

Close the ~5-order calibration gap between SCAPS PDF direct N_t values
(e.g. PVK/ETL `N_t_cm2 = 1e13`) and SolarLab's E1.5 effective N_t
(`N_t_cm2 = 1e8` empirical). Once closed:

- `configs/scaps_mirror.yaml` carries SCAPS direct N_t values
- `scripts/run_scaps_validation.py` drops `_INTERFACE_DEFECT_N_T_CALIBRATION = 1e-4`
- ETL doping over-sensitivity (1075 vs SCAPS 137 mV) auto-collapses because
  the new face densities don't track bulk N_D_ETL linearly
- Bulk N_t sweep (currently 0 mV, masked by interface SRH dominance)
  responds to N_t — interface SRH no longer 250× larger than bulk

## Background — what E1.5 does today

`solver/mol.py:_apply_interface_recombination` evaluates the SRH rate as

    R_int = (n_eval · p_eval − ni²_eff) /
            [(p_eval + p1)/v_n + (n_eval + n1)/v_p]

with

- `n_eval = n[idx + 1]`  (ETL interior bulk node, post-mask)
- `p_eval = p[idx − 1]`  (PVK interior bulk node)
- `ni²_eff = n_R_eq · p_L_eq`  (detailed-balance dark-equilibrium product)
- `v_n, v_p` from `DeviceStack.interfaces[k]`
- `n1, p1` from `srh_n1_p1_from_trap_depth(E_t)` on the reference side

Cross-carrier sampling (`n` from R, `p` from L) is the proven path that
flipped the 2 CBO trend xfails GREEN at Phase E1.5 — direction matches
SCAPS' monotonic cliff→spike rise. The PROBLEM is the *magnitude*:
ETL bulk n at the interior node is N_D_ETL (1e24 m⁻³) where SCAPS' face
plane density is ~5 orders smaller. Hence the empirical N_t calibration.

## Phase A probe data (2026-05-26)

`scripts/probe_interface_face_densities.py` dumped n, p, φ at
`(idx − 1, idx, idx + 1)` for three V_app points (baseline V_oc=1.069 V,
cliff ΔE_C=−0.5 V_oc=0.794 V, spike ΔE_C=+0.3 V_oc=1.075 V) on
`configs/scaps_mirror.yaml` (E1.5 active). Output in
`outputs/scaps_validation_e1_6a_probe.txt`.

Three face-density candidates compared:

| | Baseline (V_oc=1.07) | Cliff (V_oc=0.79) | Spike (V_oc=1.08) |
|---|---|---|---|
| **E1.5 cross-carrier** | n·p = 7.4e46 | 3.0e47 (4× bigger) | 7.6e41 (5 orders smaller) |
| **Boltzmann-from-Fermi (E1.6 attempt-2)** | -5.6e54 (NEGATIVE — unphysical) | -1.7e61 (unphysical) | -3.6e39 |
| **SG-Selberherr** | -3.5e40 (NEGATIVE because p_R noise) | -3.1e40 (UNCHANGED) | 3.3e38 |

**Key observation:** SG-Selberherr face density does NOT capture cliff
direction. The np product is ~constant between baseline and cliff
because the formula collapses on the ETL-bulk side where p is at
machine epsilon (numerical noise around ni²_ETL/N_D_ETL ≈ 5.8e−5 m⁻³).
The negative values reflect that machine epsilon, not actual physics.

**E1.5 cross-carrier IS the right cliff-direction-capturing formula.**
Its magnitude error is a separate problem.

## Phase B options (revised after Phase A)

### Option B-1 (REJECTED): SG-Selberherr face density extraction

Extract face densities from the SG flux discretization
(`physics/continuity.py:sg_fluxes_n/p`) using the Selberherr formula

    n_face = (n_L − n_R · exp(−ξ_n)) / (1 − exp(−ξ_n))

where ξ_n = (Φ_n_R − Φ_n_L) / V_T = (φ_R + χ_R − φ_L − χ_L) / V_T.

**Why rejected:** Phase A probe shows SG-Selberherr collapses on the
n+ ETL side where p is machine epsilon. The formula doesn't capture
the cliff direction because p_R ≈ 0 always, so p_face ≈ 0 too. Cliff
direction (which requires p_L > p_R growth) is lost.

### Option B-2 (RECOMMENDED): Anderson v_eff calibration per interface

Keep E1.5 cross-carrier sampling (proven direction). Replace empirical
SRV calibration with a PHYSICALLY-DERIVED per-interface attenuation
factor computed at build time from layer parameters:

    v_eff_n[k] = v_n[k] · α_n[k]
    v_eff_p[k] = v_p[k] · α_p[k]

where α represents the "this carrier rarely reaches the trap" ratio.
Candidates for α:

| Form | α_n value | Notes |
|---|---|---|
| Anderson V_bend depletion | exp(−V_bend_R/V_T) | For ETL N_D=1e24 m⁻³ on PVK N_D=1e20 m⁻³ → V_bend_R ≈ 1 mV → α ≈ 1 (no help) |
| Diffusion-length factor | L_n / d_PVK | Ratio of electron diffusion length to PVK thickness; physical but small (~0.1) |
| Constant per-heterojunction | 1e−5 for n+/PVK | Calibrated against SCAPS direct N_t; not physically derived but explicit |
| Boltzmann face from Fermi-continuity | exp(−|ΔE_C|/V_T) · (N_C_R/N_C_L) | Phase E1.6 v2 formulation — but gave Newton crashes |

**Recommendation:** ship Option B-2 with explicit calibration factor as
a new `InterfaceDefect.calibration_factor` field defaulting to 1.0. SCAPS
PVK/ETL preset sets it to 1e−5 with reference to validation report.
Document that the factor is per-heterojunction (depends on doping
asymmetry, band offset, depletion width — no closed-form single source).

**Effort:** ~30 LoC + RED tests. Roughly 1-2 days.

**Limitations:** still empirical (calibration factor is per-stack). Same
problem as today's `_INTERFACE_DEFECT_N_T_CALIBRATION = 1e−4` in the
validation script, just moved from script-side to data-model. Honest
about the gap.

### Option B-3 (research-grade): thin-shell volumetric SRH

Treat interface defect as N_t [cm⁻³] over a δ = 1 nm shell, integrate
bulk SRH formula over the shell:

    R_int = ∫_{δ shell} R_bulk_SRH(n(x), p(x), τ(x)) dx

where τ(x) = 1 / (σ · v_th · N_t_vol). The bulk-SRH machinery already
handles interface-plane n, p correctly via the SG flux discretization.

**Why interesting:** SCAPS-equivalent N_t_cm2 = N_t_vol · δ. With
δ = 1 nm and SCAPS N_t_cm2 = 1e13, N_t_vol = 1e20 cm⁻³ = 1e26 m⁻³.
Bulk τ = 1 / (1e−19 · 1e5 · 1e26) = 1e−12 s — extreme value. Solver
stiffness implications need testing.

**Effort:** ~150-300 LoC + thorough RED tests + numerical validation.
Multi-week. Higher risk than B-2 but truly physics-grounded.

**Status:** consider for Phase E2 (post-partner) if B-2 calibration
proves insufficient over multiple presets.

## Recommendation summary

Recommend **Option B-2** (Anderson v_eff calibration). Rationale:

1. Phase A probe ruled out the SG-flux-extraction approach (B-1).
2. B-2 keeps E1.5's proven cliff direction.
3. B-2 makes the calibration EXPLICIT in the data model rather than
   hidden in a validation-script multiplier. Partner can see the
   calibration factor in the YAML and decide if it's acceptable.
4. B-2 is 30 LoC vs B-3's multi-week refactor — ships fast.
5. B-3 stays available as Phase E2 follow-on if partner pushes for
   physically-grounded face densities.

## Acceptance criteria for Option B-2

- `InterfaceDefect.calibration_factor: float = 1.0` field added
- `solver/mol.py:_apply_interface_recombination` multiplies v_n, v_p by the
  calibration factor before computing R_int
- `scaps_compat/loader.py` parses `calibration_factor` from YAML
  `interfaces:` block
- `configs/scaps_mirror.yaml` updated: `N_t_cm2: 1e13` (SCAPS direct) +
  `calibration_factor: 1e-5`
- `scripts/run_scaps_validation.py` drops `_INTERFACE_DEFECT_N_T_CALIBRATION`
- Test gates: baseline V_oc stays in [1.05, 1.25] window, CBO sweep
  cliff direction passes, ETL doping range ≤ 250 mV across [1e16, 1e19]
- Update `docs/scaps_validation_report.md` with the calibration mapping
