# Phase G — base V<sub>oc</sub> audit (Phase E1.14, 2026-05-26)

**Status:** investigation complete, no code change
**Branch:** `e1.14-base-voc-audit` (closing without commit)

## Goal

Close the 99 mV base J-V V<sub>oc</sub> gap (SolarLab 1.069 V vs SCAPS PDF baseline
1.168 V) by auditing scaps_mirror.yaml MAPbI3 absorber recombination
parameters (B<sub>rad</sub>, C<sub>n</sub>, C<sub>p</sub>, τ_n, τ_p) against SCAPS PDF reference.

## Method

1. Strip PVK/ETL interface defect via `dataclasses.replace` (interfaces=(),
   interface_defects=()) — isolates bulk recombination contribution.
2. Sweep tier (full / fast / legacy) — checks per-RHS upgrades (radiative
   reabsorption, Robin contacts, μ(E)) influence.
3. Recombination channel sensitivity — kill SRH (τ→1 s), B<sub>rad</sub> (→0), or
   Auger (C<sub>n</sub>=C<sub>p</sub>=0) individually + together.

## Results

### Tier sensitivity (no interface defect)

| Mode | V<sub>oc</sub> | Notes |
|---|---|---|
| FULL | 1.0905 V | reference (radiative reabsorption + Robin off because no v_sat / S on layers) |
| FAST | 1.0905 V | identical |
| LEGACY | 0.0000 V | solver fails (chi/Eg-aware path needed for scaps_mirror) |

**Per-RHS upgrades contribute 0 mV** to base V<sub>oc</sub> (none of v_sat / S_*
fields populated in scaps_mirror.yaml, so the upgrades are no-ops by
the "enable-if-configured" tier-as-ceiling rule).

### Recombination channel sensitivity

Baseline V<sub>oc</sub> = 1.0905 V (full tier, no interface defect).

| Channel removed | V<sub>oc</sub> | ΔV<sub>oc</sub> |
|---|---|---|
| Bulk SRH (τ → 1 s) | 1.0912 V | **+1 mV** |
| Radiative (B<sub>rad</sub> → 0) | 1.0980 V | **+7 mV** |
| Auger (C<sub>n</sub> = C<sub>p</sub> → 0) | 1.1064 V | **+16 mV** |
| All three together | 0.0000 V | solver fails — no J<sub>0</sub> source |

**Combined bulk recombination accounts for only ~25 mV** of the
99 mV base-J-V gap. The remaining ~74 mV is STRUCTURAL.

## Diagnosis

The 99 mV gap is NOT a parameter calibration issue. Likely sources:

1. **J<sub>sc</sub> shortfall** (24.0 vs SCAPS 26.3 mA/cm², ΔJ<sub>sc</sub> 10 %). TMM
   Fresnel reflection at spiro/MAPbI3 trims the SCAPS scalar-α
   absorption integral. Contributes ~2-3 mV V<sub>oc</sub> via the
   ``ln(J<sub>sc</sub> / J<sub>0</sub>)`` term — small, but real.
2. **Boundary-condition convention.** SolarLab uses ohmic Dirichlet pins
   at the outer contacts (n_L / n_R / p_L / p_R from layer
   equilibrium); SCAPS' contact model has configurable Φ<sub>b</sub>
   (barrier height) and majority-carrier-only conduction. Different
   contact J<sub>0</sub>.
3. **SCAPS may use the Boltzmann statistics extension** for degenerate
   carriers; SolarLab is strictly non-degenerate Boltzmann. ETL N<sub>D</sub> =
   1e18 cm⁻³ is near degenerate; effective ni² may differ.
4. **Detailed-balance B<sub>rad</sub> scaling under photon recycling.** FAST tier
   applies the build-time ``B<sub>rad</sub> *= P<sub>esc</sub>`` scaling per Phase 3.1.
   SCAPS uses the bulk B<sub>rad</sub> directly. Net B<sub>rad</sub> in our solver is
   smaller than SCAPS' input (which would RAISE V<sub>oc</sub> — wrong direction).

None of these are tractable without (a) the exact SCAPS source-code path,
or (b) running SCAPS itself on identical params and comparing intermediate
state. Both out of scope for a half-day audit.

## Decision

**Defer base V<sub>oc</sub> closure indefinitely.** The 99 mV gap is acceptable
parity (10 % envelope on the metric, matches the
``test_scaps_mirror_baseline.py`` window of [1.05, 1.25] V). Partner
review of the validation report will decide if closer match is required
(would unlock further investigation budget).

Skip to **Step 6 (Phase F) — PVK doping direction**. Sensitivity sweep
shows PVK doping range matches SCAPS (34 mV) but direction is reversed —
likely an HTL/PVK band-bending / interface-defect issue independent of
base V<sub>oc</sub>.
